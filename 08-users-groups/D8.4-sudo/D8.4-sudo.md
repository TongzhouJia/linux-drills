# KP D8.4 — Configure Superuser Access (sudo / sudoers)
**6 tasks · RHEL 10 · server1 as root · always edit with visudo**

**1.** Create a sudo **command alias** named `MESSAGES` for the command `/bin/tail -f /var/log/messages`.

**2.** Grant the `dba_managers` group **full** sudo access (all commands).

**3.** Grant the `dba_intern` group permission to run **only** the `MESSAGES` alias via sudo.

**4.** Configure user `manny` to run `/usr/bin/systemctl` via sudo **without being prompted for a password** (NOPASSWD).

**5.** Confirm the `wheel` group has full sudo access (and add a user to it to test).

**6.** Create the rule as a **drop-in file** under `/etc/sudoers.d/` (edited safely), not by editing the main sudoers file directly.
