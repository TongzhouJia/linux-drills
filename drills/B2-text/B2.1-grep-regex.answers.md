# KP B2.1 答案 — grep & Regular Expressions

> 全部在真实 bash + 样例文件上跑过、输出核对无误。心法：**`-o` 抽取、`-E` 用扩展正则、`^$` 锚定、`-c` 数数、`-v` 取反**。

---

**1.** 计数匹配行。
```bash
grep -c 'Failed password' syslog.log      # → 4
```
*为什么：* `-c` 只输出匹配的**行数**（不是匹配次数）。数"失败登录多少次"最常用。

**2.** 忽略大小写。
```bash
grep -i 'error' syslog.log
# → Jul 17 10:09:10 server1 kernel: EXT4-fs error (device sda1) in ext4_reserve
```
*为什么：* `-i` 让匹配无视大小写，`Error/ERROR/error` 都命中。排障时防止漏掉不同写法。

**3.** 反向匹配 + 计数。
```bash
grep -v 'sshd' syslog.log | wc -l         # → 8
```
*为什么：* `-v` 输出**不匹配**的行。"排除某类噪音再统计"套路。

**4.** 带行号。
```bash
grep -n 'kernel' syslog.log
# → 4:...Out of memory...   11:...EXT4-fs error...
```
*为什么：* `-n` 前缀行号，方便定位到原文件第几行。

**5.** 整词匹配。
```bash
grep -w 'root' passwd.sample
# → root:x:0:0:root:/root:/bin/bash
```
*为什么：* `-w` 要求 `root` 前后是**词边界**，能避免匹配到 `chroot`、`rootkit` 之类。

**6.** 抽取所有 IP（`-o` + ERE）。
```bash
grep -oE '([0-9]{1,3}\.){3}[0-9]{1,3}' syslog.log | sort -u
# → 10.0.0.1 / 10.0.0.2 / 10.0.0.3 / 10.0.0.5 / 198.51.100.9 / 203.0.113.7
```
*为什么：* `-o` 只打印**匹配的部分**（而非整行），把 IP 从日志里抠出来；`([0-9]{1,3}\.){3}[0-9]{1,3}` = 三段"数字."加最后一段数字。`sort -u` 去重。这是"提取字段/指标"的通用手法。

**7.** 行首锚 `^`。
```bash
grep -c '^svc' passwd.sample              # → 2
```
*为什么：* `^svc` 只匹配**以** `svc` 开头的行（`svc_web`、`svc_db`）。

**8.** 行尾锚 `$`。
```bash
grep '/bin/bash$' passwd.sample
# → root, tongzhou, alice, dave
```
*为什么：* `/bin/bash$` 要求 `/bin/bash` 出现在**行尾**，精确筛出 shell 是 bash 的账号（避免匹配到把 bash 写在中间的行）。

**9.** ERE 择一 `(a|b)`。
```bash
grep -E ' (404|500) ' access.log | wc -l  # → 8
```
*为什么：* `-E` 开扩展正则，`(404|500)` = 二选一；前后空格保证匹配的是**状态码字段**而不是路径/字节里的数字。（404 有 5 条、500 有 3 条，共 8。）

**10.** 上下文 `-B`（before）。
```bash
grep -B1 'Killed process' syslog.log
# → cron ... backup.sh
#   kernel ... Out of memory: Killed process 3000 (java)
```
*为什么：* `-B1` 多显示匹配行**前** 1 行；`-A` 后、`-C` 前后。看报错时的上下文（前因后果）非常实用。

---

## 一句话总结（B2.1 核心）
- **抽取**用 `-o`（配 `-E` 写正则）；**计数** `-c`；**取反** `-v`；**行号** `-n`；**整词** `-w`；**忽略大小写** `-i`。
- **锚定**：`^` 行首、`$` 行尾；**择一** `(a|b)` 需 `-E`；字符类 `[0-9]`/`[[:digit:]]`。
- **上下文** `-A/-B/-C` 看报错前后。默认 BRE，遇到 `+?(){}|` 直接上 `-E` 最省心。
