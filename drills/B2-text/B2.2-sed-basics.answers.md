# KP B2.2 答案 — sed Basics

> 全部真机跑过、输出核对无误。心法：**`sed '地址 命令'`**；最常用命令 `s`(替换)、`d`(删行)、`p`(打印，配 `-n`)。

---

**1.** 替换（每行第一处）。
```bash
sed 's#/bin/bash#/bin/zsh#' passwd.sample | grep -c '/bin/zsh'   # → 4
```
*为什么：* `s/找/换/` 默认只换**每行第一个**匹配。这里用 `#` 当定界符，省得给路径里的 `/` 转义。

**2.** 全局替换 `/g`。
```bash
sed 's/,/|/g' sales.csv | head -2
# → date|region|product|units|price
#   2026-07-01|East|Widget|10|2.50
```
*为什么：* 加 `g`(global) 才换**整行所有**匹配，否则每行只换第一个。

**3.** 第 N 处 `/2`。
```bash
echo "1 2 3 4" | sed 's/ /_/2'      # → 1 2_3 4
```
*为什么：* 数字标志只替换第 N 个匹配。`s/ /_/2g` 则表示"从第 2 个起全部替换"。

**4.** 地址范围 + `-n p`。
```bash
sed -n '2,4p' passwd.sample          # → 第 2~4 行
```
*为什么：* `-n` 关闭默认输出，`p` 只打印指定行。`2,4` 是行号范围。等价于 `head -4 | tail -3`，但更直接。

**5.** 删行 `d`（去表头）。
```bash
sed '1d' sales.csv | head -2         # 从第一条数据开始
```
*为什么：* `1d` 删第 1 行（表头）。处理 CSV 前去表头常用。`$d` 删最后一行、`2,5d` 删一段。

**6.** 按模式删行 `/pat/d`。
```bash
sed '/nologin/d' passwd.sample | wc -l   # → 5
```
*为什么：* 地址也可以是**正则** `/nologin/`，匹配的行执行 `d`。这里删掉所有系统/服务账号，只剩 5 个真实登录账号。

**7.** 末行地址 `$`。
```bash
sed -n '$p' passwd.sample            # → dave 那行
```
*为什么：* `$` = 最后一行。`sed -n '$p'` 等价 `tail -1`。

**8.** 就地修改 `-i`（带备份）。
```bash
cp passwd.sample /tmp/p
sed -i.bak 's/tongzhou/tzhou/g' /tmp/p
#   /tmp/p 已改；/tmp/p.bak 是原文件
```
*为什么：* `-i` **直接写回文件**（危险！）。`-i.bak` 会先把原文件存成 `.bak`。**没把握就先不加 `-i`，看屏幕输出确认无误再加**——这是运维改配置的安全习惯。

**9.** 换定界符处理路径。
```bash
sed 's#/home#/users#g' passwd.sample | grep /users | head -2
# → …:/users/tongzhou:…   …:/users/alice:…
```
*为什么：* `s` 后面第一个字符就是定界符，可用 `#` `|` `,` 等。改带 `/` 的路径时换成 `#`，避免满屏 `\/`。

**10.** `&` 代表整个匹配。
```bash
echo "error 500" | sed 's/500/[&]/'  # → error [500]
```
*为什么：* 替换文本里的 `&` = 刚匹配到的内容。给匹配加前后缀（高亮、打标签）很方便，如 `s/[0-9]\+/<&>/g`。

---

## 一句话总结（B2.2 核心）
- **`s/找/换/`** 默认每行一次，`g` 全部、`N` 第 N 个、`i` 忽略大小写；定界符可换（`s#…#…#`）。
- **地址**决定改哪行：行号 `3`、范围 `2,4`、末行 `$`、正则 `/pat/`；`-n` + `p` = 只打印，`d` = 删行。
- **`-i` 就地改**很危险，练习用副本、生产用 `-i.bak`；`&` 引用整个匹配。
