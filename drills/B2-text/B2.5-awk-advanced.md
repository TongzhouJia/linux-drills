# KP B2.5 — awk Advanced
**`BEGIN`/`END` · 累加求和/平均 · 计算字段 · `if/else` · 正则匹配 `~` · `FS`/`OFS` · min/max**

**10 tasks · ⭐⭐ · RHEL 10 · server1 · run each against the sample files and match the result**

> 样例在 `samples/`。这一节让 awk "会算账"：`BEGIN` 开场初始化、逐行累加、`END` 收尾出结果。求和、平均、分类计数、算营收——运维出指标的核心套路。

---

**1.** Sum the total bytes served (last field) across all `access.log` lines.
*Acceptance:* `awk '{s+=$NF} END{print s}'` → `23936`.

**2.** Print the **average** bytes per request, to 1 decimal.
*Acceptance:* `awk '{s+=$NF; n++} END{printf "%.1f\n", s/n}'` → `797.9`.

**3.** From `sales.csv`, compute total revenue = Σ(units × price), skipping the header, to 2 decimals.
*Acceptance:* `awk -F, 'NR>1{r+=$4*$5} END{printf "%.2f\n", r}'` → `506.31`.

**4.** Count how many `access.log` requests hit a path containing `/api/`.
*Acceptance:* `awk '$0 ~ /\/api\// {c++} END{print c}'` → `10`.

**5.** Print a header line then the data, using `BEGIN`.
*Acceptance:* `awk -F, 'BEGIN{print "PRODUCT UNITS"} NR>1{print $3,$4}'` → `PRODUCT UNITS` then `Widget 10`, …

**6.** Classify each `access.log` request by status (field 9) into ok (<400), warn (400–499), err (≥500) and print the three counts.
*Acceptance:* `awk '{if($9>=500)e++; else if($9>=400)w++; else ok++} END{print "ok="ok,"warn="w,"err="e}'` → `ok=19 warn=6 err=5`.

**7.** Print each product with its per-row revenue (`units*price`), 2 decimals, aligned.
*Acceptance:* `awk -F, 'NR>1{printf "%-8s %.2f\n", $3, $4*$5}'` → `Widget   25.00`, `Gadget   49.95`, `Gizmo    42.00`, …

**8.** Set `FS` and `OFS` in `BEGIN` to reformat `passwd.sample` as `user | uid`.
*Acceptance:* `awk 'BEGIN{FS=":"; OFS=" | "} {print $1,$3}'` → `root | 0`, `daemon | 1`, …

**9.** Count the `Failed password` events in `syslog.log` with a regex pattern block.
*Acceptance:* `awk '/Failed password/{c++} END{print c}'` → `4`.

**10.** Find the largest single response size (max of the last field) in `access.log`.
*Acceptance:* `awk '$NF>max{max=$NF} END{print max}'` → `4096`.
