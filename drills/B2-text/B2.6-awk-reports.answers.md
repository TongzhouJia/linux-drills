# KP B2.6 答案 — awk Associative Arrays & Reports

> 全部真机跑过、输出核对无误。**核心心法：`arr[$key]++` 或 `arr[$key]+=值` 分组，`END{for(k in arr) ...}` 输出，再 `| sort` 排序。** 记住这套，日志分析随便出。

---

**1.** 按 IP 计数。
```bash
awk '{c[$1]++} END{for(ip in c) print c[ip], ip}' access.log | sort -rn
# → 10 10.0.0.1 / 5 10.0.0.2 / 4 203.0.113.7 / ...
```
*为什么：* `c[$1]++` 以 IP 为下标累加——**一行完成 GROUP BY**。`END` 遍历字典输出。awk 数组无序，排序交给 `sort`（把计数放前面才好 `sort -rn`）。

**2.** top-N。
```bash
awk '{c[$1]++} END{for(ip in c) print c[ip], ip}' access.log | sort -rn | head -3
```
*为什么：* 分组统计 + `sort -rn`(按数值降序) + `head -N`(取前 N) = 经典"top talkers"。"谁请求最多 / 谁报错最多"就是这个。

**3.** 按状态码计数。
```bash
awk '{c[$9]++} END{for(s in c) print s,c[s]}' access.log | sort
```
*为什么：* 把下标换成状态码字段 `$9`。同一套模板换个 key 就换个维度。

**4.** 分组求和（营收）。
```bash
awk -F, 'NR>1{r[$2]+=$4*$5} END{for(x in r) printf "%s %.2f\n",x,r[x]}' sales.csv | sort
# → East 196.96 / North 191.41 / West 117.94
```
*为什么：* `+=` 而非 `++`：按地区累加"数量×单价"。三地区合计正好 506.31（= B2.5 的总营收，自洽）。

**5.** 分组求和（销量）。
```bash
awk -F, 'NR>1{u[$3]+=$4} END{for(p in u) print p,u[p]}' sales.csv | sort
# → Gadget 19 / Gizmo 16 / Widget 37
```

**6.** 去重计数 `length(arr)`。
```bash
awk '{seen[$1]++} END{print length(seen)}' access.log   # → 6
```
*为什么：* 每个不同 IP 建一个下标，`length()` 返回**不同 key 的个数** = 去重后数量。等价 `awk '{print $1}' | sort -u | wc -l`，但一遍搞定。

**7.** 找字段再计数（暴力破解源）。
```bash
awk '/Failed password/{for(i=1;i<=NF;i++) if($i=="from") c[$(i+1)]++} END{for(x in c) print c[x],x}' syslog.log | sort -rn
# → 3 203.0.113.7 / 1 198.51.100.9
```
*为什么：* IP 的列号不固定，就**循环找到 `from` 这个词、取它后一列** `$(i+1)`。这题非常实战：一眼看出谁在爆破 SSH。

**8.** 复合键（多维分组）。
```bash
awk -F, 'NR>1{c[$2","$3]++} END{for(k in c) print k,c[k]}' sales.csv | sort
# → East,Gadget 1 / East,Gizmo 2 / ...
```
*为什么：* 把两列拼成一个下标 `$2","$3`，就实现"按(地区,产品)双维度分组"。相当于 SQL 的 `GROUP BY region, product`。

**9.** 分组求和（流量）。
```bash
awk '{b[$1]+=$NF} END{for(ip in b) print b[ip],ip}' access.log | sort -rn
# → 8256 10.0.0.2 / 5376 10.0.0.1 / ...
```
*为什么：* 注意结果与"请求数"排名**不同**：10.0.0.2 请求少但字节多（下载大文件）。分组维度不同、结论不同——这正是报表的价值。

**10.** `END` 里求最大（众数）。
```bash
awk '{c[$9]++} END{for(s in c) if(c[s]>m){m=c[s];top=s}; print top,m}' access.log   # → 200 17
```
*为什么：* 先分组计数，再在 `END` 里遍历字典找计数最大的 key。不排序也能取 top-1。

**11.** 拆时间戳按小时分桶。
```bash
awk '{split($4,a,":"); c[a[2]]++} END{for(h in c) print h,c[h]}' access.log | sort
# → 10 14 / 11 16
```
*为什么：* `split($4,a,":")` 把 `[17/Jul/2026:10:00:01` 按 `:` 拆进数组 `a`，`a[2]` 就是小时。**时间窗/按小时出趋势**的基本功。

**12.** 综合报表（计数 + 占比）。
```bash
awk '{c[$9]++; t++} END{for(s in c) printf "%s  %2d  %.1f%%\n",s,c[s],100*c[s]/t}' access.log | sort
# → 200  17  56.7% / 404   5  16.7% / 500   3  10.0% / ...
```
*为什么：* 一遍同时统计各状态计数 `c[]` 和总数 `t`，`END` 里出"码/量/占比"三列报表。这就是运维日报的样子——**面试时手写出这条，Bash 加分项直接坐实**。

---

## 一句话总结（B2.6 核心）
- **模板**：`arr[$key]++`（计数）或 `arr[$key]+=v`（求和）→ `END{for(k in arr) print ...}` → `| sort [-rn] | head`。
- **换 key 换维度**：IP、状态码、地区、`$2","$3` 复合键、拆时间戳按小时。
- **去重计数** `length(arr)`；**top-N** = 分组 + `sort -rn | head`；**众数** = END 里比较求最大。
- 这一节 = "把一堆日志/记录变成一张指标表"，是运维 Bash 最硬的招牌功夫。
