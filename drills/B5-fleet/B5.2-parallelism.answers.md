# KP B5.2 答案 — Parallelism

> 全部真机跑过、输出核对无误（串行 3s vs 并行 1s，实测提速）。心法：**`&` 把任务丢后台、`wait` 等回来；量大用 `xargs -P N` 控并发。**

---

**1.** 后台 + `wait`。
```bash
for i in 1 2 3; do ( sleep 0.3; echo "job $i done" ) & done
wait
echo "all done"
```
*为什么：* `cmd &` 把命令丢到后台立即返回，循环一下起 3 个；`wait`（无参）**等所有后台子进程结束**再往下。注意输出顺序不定（并发）。

**2.** 串行 vs 并行提速。
```bash
SECONDS=0; for i in 1 2 3; do sleep 1; done;        echo "serial: ${SECONDS}s"   # → 3s
SECONDS=0; for i in 1 2 3; do sleep 1 & done; wait; echo "parallel: ${SECONDS}s" # → 1s
```
*为什么：* 串行 3×1=3 秒；并行同时开跑、总耗时≈最慢的一个=1 秒。`SECONDS` 是 bash 内建秒表（赋 0 开始计）。这就是并行的意义。

**3–4. `xargs -P` 并行 + `-I{}`。**
```bash
printf '%s\n' alpha bravo charlie delta | xargs -P4 -I{} sh -c 'echo processed {}'
```
*为什么：* `xargs -P4` = 最多 **4 个并发**；`-I{}` 把每行输入替换到 `{}` 位置。比手写 `&`+`wait` 简洁，还自动控并发。是并行处理"一堆输入"的首选。

**5.** 控制并发数。
```bash
seq 1 4 | xargs -P2 -I{} sh -c 'echo "start {}"; sleep 0.2; echo "end {}"'
# start 1 / start 2 （先俩）→ end… → start 3 / start 4
```
*为什么：* `-P2` 限制同时最多 2 个。**并发数不是越大越好**——太多会把 CPU/网络/被访问的服务器压垮。批量 ssh 常设 `-P 10~20`。

**6.** 取后台任务的退出码。
```bash
( exit 3 ) & pid=$!
wait "$pid"; echo "$?"        # → 3
```
*为什么：* `$!` 是**最近一个后台任务的 PID**；`wait $pid` 等它并**返回它的退出码**。这样并行跑完还能知道每个任务成没成功。

**7.** 手动控并发（每 N 个 drain 一次）。
```bash
max=2; n=0
for job in A B C D E; do
  ( sleep 0.2; echo "done $job" ) &
  (( ++n % max == 0 )) && wait     # 每起 2 个就 wait 一批
done
wait                               # 收尾剩余
```
*为什么：* 不想用 xargs 时，用"起够 N 个就 `wait` 一批"手动限并发。适合循环体逻辑复杂、不好塞进 xargs 的场景。（更精细可用 `wait -n` 一有完成就补一个。）

**8.** 并行处理文件。
```bash
ls *.log *.csv | xargs -P4 -I{} sh -c 'printf "%-14s %s\n" "{}" "$(wc -l < "{}")"'
```
*为什么：* 把"对每个文件做处理"并行化。真实场景：并行压缩一堆日志、并行校验一批文件、并行对多台机器 ssh（`xargs -P10 -I{} ssh {} uptime`）。

---

## 一句话总结（B5.2 核心）
- **手动并发**：`cmd &` 丢后台、`wait` 等全部、`wait $!`（`$!`=后台PID）取单个退出码。
- **批量并行**：`… | xargs -P N -I{} cmd {}`，`-P N` 控并发数（别设太大压垮机器）。
- **提速**：并行总耗时≈最慢的一个；串行是各任务之和。
- 手动限并发：起够 N 个就 `wait` 一批。
