# KP B5.4 答案 — jq for JSON

> 全部真机跑过、输出核对无误。心法：**`jq '过滤器' file`**——`.` 取字段、`[]` 展开数组、`select()` 过滤、`-r` 出裸文本（去引号）。

---

**1.** 取顶层字段。
```bash
jq -r '.datacenter' servers.json      # → dc1
```
*为什么：* `.key` 访问字段。**`-r`（raw）去掉外层引号**——要把值喂给别的命令时几乎总要加 `-r`，否则带引号。

**2.** 数组长度。
```bash
jq '.servers | length'                # → 4
```
*为什么：* `|` 是 jq 的管道（把左边结果传给右边），`length` 求数组/对象/字符串长度。

**3.** 展开数组取字段。
```bash
jq -r '.servers[].name'               # → web1 web2 db1 cache1
```
*为什么：* `.servers[]` 把数组**逐个元素**吐出，再 `.name` 取每个的字段。这是遍历 JSON 数组的基本式。

**4–5. `select` 过滤。**
```bash
jq -r '.servers[] | select(.cpu>50) | .name'          # → web2
jq -r '.servers[] | select(.status=="down") | .name'  # → db1
```
*为什么：* `select(条件)` 只保留满足条件的元素——相当于 awk 的 pattern。数值 `>`、字符串 `==`。找"CPU 超标的机器""挂掉的服务"就这么筛。

**6.** 字符串插值。
```bash
jq -r '.servers[] | "\(.name): \(.status)"'
# → web1: up / web2: up / db1: down / cache1: up
```
*为什么：* `"\(表达式)"` 把字段插进字符串（类似 bash 的 `${}`）。把 JSON 变成人类可读的一行行。

**7.** 转 CSV。
```bash
jq -r '.servers[] | [.name,.cpu,.status] | @csv'
# → "web1",45,"up" ...
```
*为什么：* 先 `[字段列表]` 组成数组，再 `@csv` 转 CSV（自动加引号转义）；`@tsv` 出 Tab 分隔。**把 API 的 JSON 落成表格**喂给 awk/Excel，天天用。

**8.** 从 bash 变量造 JSON。
```bash
jq -n --arg name web5 --argjson cpu 20 '{name:$name, cpu:$cpu, status:"up"}'
# → { "name": "web5", "cpu": 20, "status": "up" }
```
*为什么：* `-n`（null 输入，凭空造）；`--arg` 传**字符串**变量、`--argjson` 传**数字/布尔/JSON**（不加引号）。**给 API 发请求体**时用它安全地拼 JSON（比手写字符串拼接不容易出转义 bug）。

**Bonus：平均值 + 真实场景。**
```bash
jq '[.servers[].cpu] | add/length'                # → 45   （取所有cpu成数组，求和/个数）
curl -s https://api.example.com/servers | jq '.'  # 最常见：管道给 jq 处理接口返回
```

## 一句话总结（B5.4 核心）
- `jq '过滤器' file`：`.field` 取值、`.arr[]` 展开、`select(cond)` 过滤、`"\(.x)"` 插值、`@csv/@tsv` 转表；**取值给别的命令用记得 `-r`**。
- 造 JSON：`jq -n --arg s str --argjson n 20 '{...}'`（发 API 请求体）。
- 日常式：`curl -s URL | jq '...'`。jq 虽非纯 Bash，但和 Bash 是黄金搭档。
