# KP B5.3 — Scheduling Integration (cron + systemd timer)
**cron 五字段 · 日志重定向 · PATH 坑 · `.service`+`.timer` · `OnCalendar` · `list-timers` · flock+日志包装**

**8 tasks · ⭐⭐ · RHEL 10 · server1 · 衔接你 RHCSA 的 D6.1/D6.2**

> 脚本写好了，还得**定时自动跑**。cron 是传统方式，systemd timer 是 RHEL 10 主推（你 RHCSA 也强调）。关键不只是"配上时间"，还要**输出进日志、加锁防重叠、用绝对路径**——否则定时任务最爱"静默失败"。

---

**1.** Read a cron schedule: what does `30 2 * * *` mean? (fields: minute hour day-of-month month day-of-week)
*Acceptance:* "at 02:30 every day". And `*/15 * * * *` = "every 15 minutes".

**2.** Write a crontab entry (`crontab -e`) to run `/usr/local/bin/backup.sh` daily at 02:30, appending output to a log.
*Acceptance:* `30 2 * * * /usr/local/bin/backup.sh >> /var/log/backup.log 2>&1`.

**3.** The cron **PATH gotcha**: why might a script that works in your shell fail under cron, and the fix?
*Acceptance:* cron runs with a minimal `PATH` → use **absolute paths** for commands, or set `PATH=` at the top of the script/crontab.

**4.** Make sure a cron job's output is **captured** (cron emails output by default).
*Acceptance:* append `>> /var/log/job.log 2>&1` (both stdout and stderr) so nothing is lost or mailed.

**5.** Write a systemd **`.service` (Type=oneshot)** + **`.timer`** pair to run the backup at 02:30.
*Acceptance:* service `ExecStart=/usr/local/bin/backup.sh …`; timer `OnCalendar=*-*-* 02:30:00` with `Persistent=true`.

**6.** Check an `OnCalendar` expression resolves correctly with `systemd-analyze calendar`.
*Acceptance:* `systemd-analyze calendar '*-*-* 02:30:00'` → shows the next elapse time.

**7.** Enable + start the timer and list active timers.
*Acceptance:* `systemctl enable --now backup.timer`; `systemctl list-timers` shows it with a NEXT time.

**8.** Make the scheduled script safe: wrap it with `flock` (skip if a previous run is still going) and **timestamped logging**.
*Acceptance:* second concurrent tick prints `already running, skip` and exits 0; each run logs `[YYYY-MM-DD HH:MM:SS] …`.
