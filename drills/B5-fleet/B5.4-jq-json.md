# KP B5.4 — jq for JSON （可选）
**字段访问 · 数组/`length` · `select` 过滤 · 字符串插值 · `@csv` · `-r` · 从 bash 造 JSON**

**8 tasks · ⭐⭐ · RHEL 10 · server1 · run each against `samples/servers.json`**

> ⚠️ **可选节**：`jq` 是**独立工具**（不是纯 Bash），但现代运维处理 API/云平台返回的 JSON 几乎离不开它，值得会。RHEL 装：`sudo dnf install -y jq`。
> 样例 `samples/servers.json`（一个数据中心 + 4 台服务器的数组）。

---

**1.** Extract the top-level `.datacenter` field (raw, no quotes).
*Acceptance:* `jq -r '.datacenter' servers.json` → `dc1`.

**2.** Count the servers.
*Acceptance:* `jq '.servers | length'` → `4`.

**3.** List all server **names** (raw).
*Acceptance:* `jq -r '.servers[].name'` → `web1 web2 db1 cache1`.

**4.** Names of servers with **cpu > 50** (`select`).
*Acceptance:* `jq -r '.servers[] | select(.cpu>50) | .name'` → `web2`.

**5.** Names of servers whose `status` is `down`.
*Acceptance:* `jq -r '.servers[] | select(.status=="down") | .name'` → `db1`.

**6.** Format each server as `name: status` using string interpolation.
*Acceptance:* `jq -r '.servers[] | "\(.name): \(.status)"'` → `web1: up`, `web2: up`, `db1: down`, `cache1: up`.

**7.** Output selected fields as **CSV**.
*Acceptance:* `jq -r '.servers[] | [.name,.cpu,.status] | @csv'` → `"web1",45,"up"`, …

**8.** **Build** a JSON object from shell variables (`--arg`/`--argjson`).
*Acceptance:* `jq -n --arg name web5 --argjson cpu 20 '{name:$name, cpu:$cpu, status:"up"}'` → a JSON object with those values.

*Bonus:* average CPU → `jq '[.servers[].cpu] | add/length'` → `45`. And `curl -s https://api.example.com/servers | jq '.'` is the everyday pattern.
