# KP B5.1 答案 — SSH Fan-out

> 循环/收集/出错处理的**骨架**用 mock（假 ssh，令 server2 "宕机"）真机验证过；ssh 命令本身需连真实 server1/server2。心法：**免密 + 循环 + `BatchMode` 不卡住 + 出错继续 + 输出带主机名。**

---

**1.** 读主机清单（跳注释/空行）。
```bash
grep -vE '^\s*#|^\s*$' hosts.txt | while read -r h; do echo "visiting $h"; done
```
*为什么：* `grep -vE '^\s*#|^\s*$'` 去掉 `#` 注释行和空行——清单文件常有这两种。

**2.** 逐台跑命令。
```bash
while read -r h; do ssh "$h" uptime; done < <(grep -vE '^\s*#|^\s*$' hosts.txt)
```
*为什么：* `ssh host cmd` 在远端执行 `cmd` 并把输出传回本地。用 `< <(...)` 喂列表（别 `cat | while`，B3.3 的子shell坑）。

**3–4 / 8. 出错继续 + 前缀 + 计数（核心骨架）。**
```bash
ok=0; fail=0
while read -r h; do
  if out=$(ssh -o BatchMode=yes -o ConnectTimeout=5 "$h" uptime 2>/dev/null); then
    echo "$out" | sed "s/^/[$h] /"; ((ok++))
  else
    echo "[$h] UNREACHABLE (rc=$?)"; ((fail++))
  fi
done < <(grep -vE '^\s*#|^\s*$' hosts.txt)
echo "reachable=$ok / $((ok+fail))"
```
输出（mock，server2 宕）：`[server1] load…` / `[server2] UNREACHABLE (rc=255)` / `[repo] load…` / `reachable=2 / 3`。
*为什么：*
- **别开 `set -e`**（或把 ssh 放 `if` 里）：某台连不上返回非 0 是**预期**，不能让整个巡检中止。
- `sed "s/^/[$h] /"` 给每行加主机名前缀，几十台的输出混在一起也认得出谁是谁。
- 累加 `ok`/`fail` 出汇总。

**5.** 脚本安全 ssh 选项。
```bash
ssh -o BatchMode=yes -o ConnectTimeout=5 "$h" true
```
*为什么：* `BatchMode=yes` = **绝不弹密码/确认提示**（没配好密钥就直接失败，而不是卡住等输入）；`ConnectTimeout=5` = 连不上 5 秒就放弃，不让一台死机拖垮整批。自动化里这俩几乎必加。

**6.** 把本地脚本丢到远端跑。
```bash
ssh "$h" 'bash -s' < local.sh          # 不用先 scp
```
*为什么：* `bash -s` 从 stdin 读脚本，`< local.sh` 把本地脚本喂过去——**远端不落文件**就执行。批量巡检/一次性修复很方便。

**7.** 配免密（前提）。
```bash
ssh-keygen -t ed25519          # 本地生成密钥对（一次）
ssh-copy-id user@server2       # 把公钥装到远端（每台一次）
ssh server2                    # 之后不再要密码
```
*为什么：* 批量 ssh 的前提是**免密**，否则循环里每台都卡着要密码。（衔接你 RHCSA 的 D9.3 SSH 密钥认证。）

**9–10. fan-out 脚本。**
```bash
#!/usr/bin/env bash
set -uo pipefail
hosts=${1:?usage: fanout.sh <hostfile> <cmd...>}; shift
rc=0
while read -r h; do
  [[ $h =~ ^[[:space:]]*# || -z ${h// } ]] && continue
  if ssh -o BatchMode=yes -o ConnectTimeout=5 "$h" "$@" 2>/dev/null | sed "s/^/[$h] /"; then :; else
    echo "[$h] FAILED"; rc=1
  fi
done < "$hosts"
exit $rc
```
用法：`./fanout.sh hosts.txt uptime`、`./fanout.sh hosts.txt 'df -P /'`。
*为什么：* `shift` 后 `"$@"` 是要在每台跑的命令（可带参数）。有一台失败 `rc=1`、最后 `exit $rc`，让调用方知道"这批不是全绿"。这就是最朴素的批量运维工具（再往上就是 Ansible，但你要的是纯 Bash）。

---

## 一句话总结（B5.1 核心）
- **前提免密**：`ssh-keygen` + `ssh-copy-id`。
- **循环骨架**：`while read h < <(grep -v 注释 hosts)`，`ssh -o BatchMode=yes -o ConnectTimeout=5 "$h" cmd`。
- **别中断**：ssh 放 `if` 里、别开 `set -e`；输出 `sed "s/^/[$h] /"` 加主机名；累计 ok/fail、`exit $rc`。
- `ssh h 'bash -s' < local.sh` 远端跑本地脚本。
