# KP B6.2 — Read & Fix Broken Scripts ⭐⭐⭐
**面试高频题型：给你一段坏脚本，找出 bug 并修好**

**12 tasks · ⭐⭐⭐ · RHEL 10 · server1 · for each snippet: say what's wrong, why, and fix it**

> 这一节练的是"读别人的脚本、一眼看出坑"。每题都是**真能复现的经典 bug**（答案里附了坏/好两种行为的实测）。先自己判断，再对答案。

---

**1.** Why does this fail when the file exists?
```bash
f="my report.txt"
if [ -f $f ]; then echo found; fi
```

**2.** Why does this print `count: command not found`?
```bash
count = 0
```

**3.** Why does this crash when `$answer` is empty?
```bash
if [ $answer = yes ]; then echo ok; fi
```

**4.** With `a=9; b=100`, why does this say `a>b` is true (and create a file named `100`)?
```bash
if [ $a > $b ]; then echo "a>b"; fi
```

**5.** Why does this print each **word** instead of each **line**?
```bash
for line in $(cat lines.txt); do echo "[$line]"; done
```

**6.** Why is this dangerous if `$dir` doesn't exist?
```bash
cd $dir
rm -f *
```

**7.** What's the syntax error here, and how would you catch it before running?
```bash
for i in 1 2 3; do
  echo $i
```

**8.** With files in the current dir, why does this print filenames instead of `files: *`?
```bash
msg="files: *"
echo $msg
```

**9.** Why does this error with `integer expression expected`?
```bash
name=root
if [ "$name" -eq root ]; then echo match; fi
```

**10.** Why is `count` still `0` after the loop?
```bash
count=0
cat items.txt | while read -r x; do ((count++)); done
echo "$count"
```

**11.** With `n=5`, why does this error instead of printing `n is 5`?
```bash
if [ n -eq 5 ]; then echo "n is 5"; fi
```

**12.** Why don't the error messages end up in `log`?
```bash
somecmd 2>&1 > log
```
