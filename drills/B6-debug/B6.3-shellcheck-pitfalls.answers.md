# KP B6.3 答案 — shellcheck & Common Pitfalls

> 心法：**每个脚本都过一遍 `shellcheck`**，它把词拆分、引号、`cd` 未检查这些坑自动标出来，比人眼可靠。下面每条都是它最常报的警告。

> 安装（RHEL/EPEL）：`sudo dnf install -y epel-release && sudo dnf install -y ShellCheck` → `shellcheck script.sh`。也可在 https://www.shellcheck.net 在线粘贴检查。

---

**1. SC2086 — 未加引号的变量（词拆分/通配）。**
```bash
for f in $files; do cp $f /backup; done      # ✗
for f in "$files"; do cp "$f" /backup; done  # 修：给 $f 加引号
```
- 问题：`$f` 不加引号，带空格的文件名会被拆成多个参数、含 `*` 会被通配。
- 更好：别 `for f in $files`，直接遍历文件 `for f in *.log` 或用数组。**最高频警告，见到必修。**

**2. SC2046 — 未加引号的命令替换。**
```bash
rm $(find . -name '*.tmp')                    # ✗ 文件名带空格就炸
find . -name '*.tmp' -delete                  # 修（首选）
# 或： find . -name '*.tmp' -exec rm {} +
```
- 问题：`$(find …)` 的输出被词拆分，带空格/换行的路径会被拆错，误删或报错。
- 修：让 `find` 自己删（`-delete` / `-exec … +`），不经过 shell 拆分。

**3. SC2006 — 老式反引号。**
```bash
d=`date +%F`        # ✗ 旧语法，不能嵌套、易读性差
d=$(date +%F)       # 修
```
- 问题：反引号 `` ` `` 是过时写法，嵌套困难、转义混乱。
- 修：一律用 `$( )`，可嵌套、更清晰。

**4. SC2164 — `cd` 不检查返回值。**
```bash
cd /var/log          # ✗ 若失败，后面在错误目录里执行
rm -f *.old
# 修：
cd /var/log || exit 1
rm -f *.old
```
- 问题：`cd` 失败不会停，接下来的 `rm` 可能在**错误目录**里跑（B6.2 Bug 6 同款）。
- 修：`cd … || exit`（或 `|| return`）。

**5. SC2115 — 变量为空时 `rm -rf` 变成删根。**
```bash
rm -rf "$TARGET/"          # ✗ 若 TARGET 为空 → rm -rf "/" 😱
rm -rf "${TARGET:?TARGET is empty}/"   # 修：空则报错退出
```
- 问题：`$TARGET` 为空时展开成 `rm -rf "/"`——灾难。
- 修：用 `${TARGET:?消息}`，变量空/未设就报错中止，绝不往下删。**涉及 `rm -rf` + 变量路径，务必加这个护栏。**

**6. SC2181 — 用 `$?` 判断上一条命令（反模式）。**
```bash
grep -q root /etc/passwd
if [ $? -ne 0 ]; then echo "no root"; fi    # ✗ 脆弱
# 修：直接把命令放进 if
if ! grep -q root /etc/passwd; then echo "no root"; fi
```
- 问题：`$? ` 容易被中间插入的命令（哪怕一句 `echo`）覆盖，判断错对象。
- 修：**直接 `if cmd` / `if ! cmd`**，别绕 `$?`。

**7. SC2004 — 算术里多余的 `$`。**
```bash
n=$(( $x + 1 ))     # ✗ $(( )) 里变量不用加 $
n=$(( x + 1 ))      # 修
```
- 问题：`$(( ))` 内部会自动解引用变量，写 `$x` 多余（虽不报错但不规范）。
- 修：直接写变量名 `x`。

**8. SC2148 — 缺 shebang。**
```bash
echo "starting backup"          # ✗ 文件开头没有 #!
# 修：第一行加
#!/usr/bin/env bash
echo "starting backup"
```
- 问题：没有 `#!` 行，脚本用哪个解释器不确定（可能被 `sh` 跑，行为不同）。
- 修：首行加 `#!/usr/bin/env bash`（或 `#!/bin/bash`）。

## 一句话总结（B6.3 核心）
- **写完脚本 `shellcheck` 过一遍**，自动抓 B6.2 那些坑。
- 最高频：**SC2086/SC2046 加引号**、**SC2164 `cd||exit`**、**SC2115 `${var:?}` 护栏**、**SC2181 直接 `if cmd`**、**SC2006 用 `$()`**、**SC2148 加 shebang**。
- 把 shellcheck 当"上线前的自动 code review"，能省掉一大半线上事故。
