# KP B6.2 答案 — Read & Fix Broken Scripts

> 每题坏/好两种行为都真机实测过。心法：**80% 的 bash bug 来自"没加引号 / 空格 / 数值当字符串比 / 子shell"这几类。** 见到就条件反射。

---

**1. 未加引号 + 空格文件名。**
- 坏：`[ -f $f ]` → `$f` 展开成 `my report.txt`，`[ ]` 看到三个词 → `[: my: binary operator expected`。
- 修：`if [[ -f $f ]]; then …`（或 `[ -f "$f" ]`）。
- 因：`[ ]` 会对未加引号的展开做词拆分；`[[ ]]` 不会。**变量一律加引号或用 `[[ ]]`。**

**2. `=` 两边有空格。**
- 坏：`count = 0` → shell 以为 `count` 是命令、`=` `0` 是参数 → `count: command not found`。
- 修：`count=0`。
- 因：赋值的 `=` **两边不能有空格**（这点和几乎所有语言相反）。

**3. 字符串比较遇空值崩溃。**
- 坏：`answer` 为空时 `[ $answer = yes ]` → `[ = yes ]` → `unary operator expected`（exit 2）。
- 修：`[ "$answer" = yes ]` 或 `[[ $answer == yes ]]`。
- 因：空变量不加引号会"消失"，`[ ]` 参数缺失。加引号后是 `[ "" = yes ]`，正常返回假。

**4. 在 `[ ]` 里用 `>` 比数字。**
- 坏：`[ $a > $b ]` → `>` 被当**重定向**，实际执行 `[ 9 ]`（非空=真）并创建名为 `100` 的文件 → 误报 `a>b` 且污染目录。
- 修：`(( a > b ))`（数值最自然）或 `[ "$a" -gt "$b" ]`。
- 因：`[ ]` 里 `<` `>` 是字符串比较且需转义，裸写会被 shell 当重定向。**数字比较用 `(( ))` 或 `-gt/-lt`。**

**5. `for line in $(cat file)` 按词拆。**
- 坏：`$(cat)` 的结果被按**空白（含空格）**拆分 → 一行 `alpha one` 变成 `alpha`、`one` 两次。
- 修：`while IFS= read -r line; do …; done < lines.txt`。
- 因：命令替换会做词拆分。**逐行处理永远用 `while read`，别用 `for … in $(cat)`。**

**6. `cd` 不检查就动手（危险！）。**
- 坏：`cd $dir` 失败（目录不存在/没权限）时脚本**不停**，接着 `rm -f *` 就在**当前错误目录**里删文件。
- 修：`cd "$dir" || exit 1`（或 `cd "$dir" || { echo "no dir">&2; exit 1; }`），再操作。
- 因：`cd` 失败只是返回非 0、不会自动停。**凡 `cd` 后要删/改，必须 `cd … || exit`。** 血泪教训级别。

**7. 少了 `done`（语法错误）。**
- 坏：`for … do echo $i`（缺 `done`）→ `syntax error: unexpected end of file`。
- 修：补上 `done`（`if` 补 `fi`、`case` 补 `esac`）。
- 抓法：**上线前 `bash -n script.sh`** 只查语法不执行，立刻报第几行。

**8. 未加引号的变量含通配符被展开。**
- 坏：`msg="files: *"; echo $msg` → 不加引号时 `*` 被当 glob **展开成当前目录文件名**。
- 修：`echo "$msg"`。
- 因：变量值里的 `* ? [` 在不加引号时会二次通配。**回显/传递变量一律 `"$var"`。**

**9. 用 `-eq` 比字符串。**
- 坏：`[ "$name" -eq root ]` → `-eq` 是**数值**比较，`root` 不是数字 → `integer expression expected`。
- 修：`[ "$name" = root ]`（字符串用 `=`/`==`）。
- 因：`-eq -ne -lt…` 只用于整数；字符串相等用 `=`（`[ ]`）或 `==`（`[[ ]]`）。

**10. 管道子shell丢变量。**
- 坏：`cat items.txt | while read …; do ((count++)); done` → 循环在**子shell**里跑，`count` 改动出了管道就没了 → 仍是 0。
- 修：`while read -r x; do ((count++)); done < items.txt`（重定向）或 `done < <(cmd)`（进程替换）。
- 因：管道每段在子shell执行，变量不回传父shell。（B1.5 / B3.3 反复强调的坑。）

**11. 测试里忘了 `$`。**
- 坏：`[ n -eq 5 ]` → 比较的是**字面字符 `n`** 而非变量值 → `integer expression expected`。
- 修：`[ "$n" -eq 5 ]`。
- 因：`[ ]` 里引用变量要写 `$n`。忘 `$` 就把变量名当字面量了。

**12. 重定向顺序错。**
- 坏：`somecmd 2>&1 > log` → 先把 stderr 指向**当前的 stdout（终端）**，再把 stdout 改到 log；错误仍留终端。
- 修：`somecmd > log 2>&1`（先定向 stdout 到文件，再让 stderr 跟随）。
- 因：重定向**从左往右**生效，`2>&1` 复制的是 stdout **当时**的指向。（B1.4 的顺序陷阱。）

---

## 一句话总结（B6.2 核心 = bug 高发区清单）
- **引号**：`"$var"`、`[[ ]]`——解决空格/空值/通配符一大票问题（Bug 1/3/8/10）。
- **赋值** `=` 两边无空格（Bug 2）；测试里别忘 `$`（Bug 11）。
- **数值比较**用 `(( ))`/`-gt`，别在 `[ ]` 用 `>`，别用 `-eq` 比字符串（Bug 4/9）。
- **逐行读**用 `while read < file`，别 `for in $(cat)`、别 `cat | while`（Bug 5/10）。
- **`cd … || exit`** 再动手（Bug 6）；**`> file 2>&1`** 顺序（Bug 12）；上线前 **`bash -n`**（Bug 7）。
