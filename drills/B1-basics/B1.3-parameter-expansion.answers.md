# KP B1.3 答案 — Parameter Expansion / 变量与参数展开

> 命令都在真实 bash 里跑过、输出核对无误。记忆锚点：**`#` 砍头、`%` 砍尾**（键盘上 `#` 在 `%` 左边，正好对应"头/左"和"尾/右"）；**单个=最短匹配，双写=最长匹配**。

---

**1.** 花括号消除歧义。
```bash
base=report
echo "${base}_v2.log"     # → report_v2.log
```
*为什么：* `$base_v2` 会被当成变量名 `base_v2`（未定义→空），得到 `.log`。`${base}` 明确框住变量名，后面的 `_v2` 才是字面。凡是变量后面紧跟字母/数字/下划线，都要加 `{}`。

**2.** `${var:-default}` 用默认值但**不赋值**。
```bash
unset LOGDIR
echo "${LOGDIR:-/var/log}"   # → /var/log
echo "${LOGDIR-UNSET}"       # → UNSET   （证明没被赋值）
```
*为什么：* `:-` 只是"这次取值时若空则用默认"，变量本身不变。适合"临时兜底"。（`:-` 带冒号=未设置或空都兜底；不带冒号 `-` 只在"未设置"时兜底。）

**3.** `${var:=default}` 用默认值**并赋值**。
```bash
unset PORT
echo "${PORT:=8080}"   # → 8080
echo "$PORT"           # → 8080   （已写回变量）
```
*为什么：* `:=` 会把默认值**写回**变量，后续都能用。常见于脚本开头给配置变量兜底：`: "${PORT:=8080}"`（前面 `:` 是空命令，只为触发展开）。
> ⚠️ 坑：`:=` 若发生在 `$( )` 子shell里（如 `x=$(echo ${Q:=99})`），赋值只在子shell生效，**不会**传回父shell。实测 `Q` 仍为空。

**4.** `${var:?message}` 必填校验。
```bash
unset USERNAME
echo "${USERNAME:?is required}"
# stderr: bash: USERNAME: is required
# 退出码非 0（脚本中会直接中止）
```
*为什么：* 变量为空/未设置时，打印错误到 **stderr** 并让脚本**非 0 退出**。写脚本做参数校验一行搞定，比 `if` 更省。

**5.** `${#var}` 取长度。
```bash
pw='S3cr3t!'
echo "${#pw}"      # → 7
```

**6.** `#` / `##` 砍**前缀**（取扩展名）。
```bash
f=archive.tar.gz
echo "${f##*.}"    # → gz       （## = 去掉最长匹配 *. → 只剩最后一段）
echo "${f#*.}"     # → tar.gz    （#  = 去掉最短匹配 *. → 去掉第一个点之前）
```
*为什么：* `#` 从**左边**砍。`*.` 是"任意字符 + 一个点"；`#` 砍最短（到第一个点），`##` 砍最长（到最后一个点）。取扩展名固定用 `${f##*.}`。

**7.** `%` / `%%` 砍**后缀**（去扩展名）。
```bash
f=report.2026.csv
echo "${f%.*}"     # → report.2026   （%  = 去掉最短的 .* → 去最后一段）
echo "${f%%.*}"    # → report        （%% = 去掉最长的 .* → 从第一个点起全砍）
```
*为什么：* `%` 从**右边**砍。去掉扩展名固定用 `${f%.*}`。

**8.** 纯展开实现 basename / dirname。
```bash
p=/var/log/nginx/access.log
echo "${p##*/}"    # → access.log       （砍到最后一个 / ）
echo "${p%/*}"     # → /var/log/nginx    （砍掉最后一个 / 及之后）
```
*为什么：* 比调用 `basename`/`dirname` 外部命令更快，循环里处理成千上万路径时差别明显。

**9.** `/` 替换一次，`//` 全替。
```bash
s=a-b-c-d
echo "${s/-/_}"    # → a_b-c-d    （只换第一个）
echo "${s//-/_}"   # → a_b_c_d    （全换）
```
*补充：* `${s/#pat/new}` 只匹配开头，`${s/%pat/new}` 只匹配结尾；`${s//pat/}`（new 留空）= 删除所有 pat。

**10.** `${var:offset:length}` 取子串（从 0 开始）。
```bash
d=2026-07-17
echo "${d:0:4}"    # → 2026   （第 0 位起 4 个）
echo "${d:5:2}"    # → 07     （第 5 位起 2 个）
```
*补充：* 负偏移取末尾要加空格或括号：`${d: -2}` 或 `${d:(-2)}` → `17`。

**11.** 大小写转换（Bash 4+）。
```bash
name=server1;  echo "${name^^}"   # → SERVER1   （^^ 全大写；^ 只首字母）
H=WEB-PROD;    echo "${H,,}"      # → web-prod   （,, 全小写；, 只首字母）
```

**12.** 综合链式处理（运维备份名解析）。
```bash
f=/data/backups/db-2026-07-17.sql.gz
base=${f##*/}          # db-2026-07-17.sql.gz     取文件名
noext=${base%.sql.gz}  # db-2026-07-17            去掉 .sql.gz
date=${noext#db-}      # 2026-07-17               去掉前缀 db-
echo "$base | $noext | $date"
```
*为什么：* 一连串 `##` `%` `#` 就把路径拆成想要的片段，全程零外部命令。这就是"日志/文件名处理八成靠展开"的意思。

---

## 一句话总结（B1.3 核心）
- **兜底**：`${v:-x}` 取默认不赋值 · `${v:=x}` 取默认并赋值 · `${v:?msg}` 空则报错退出。
- **切片**：`#`砍头/`%`砍尾，**单=最短、双=最长**；取扩展名 `${f##*.}`、去扩展名 `${f%.*}`、basename `${f##*/}`、dirname `${f%/*}`。
- **改写**：`${v/a/b}` 换一次 · `${v//a/b}` 换全部 · `${v:i:n}` 子串 · `${v^^}`/`${v,,}` 大小写。
- 运维习惯：变量后接字符一律 `${v}`，路径/文件名处理优先用展开，别动不动上 sed。
