# KP B1.2 答案 — Globbing / 文件名通配

> 命令都在真实 bash 里跑过、输出核对无误。核心心法：**glob 由 shell 在执行前展开，且只匹配真实存在的文件名**。

---

**1.** `*` = 任意长度任意字符（含空）。
```bash
ls *.log            # → access.log  app.log  error.log
```
*为什么：* `*.log` 只匹配以 `.log` **结尾**的名字。`app.log.1` 结尾是 `.1`，不匹配；`.hidden.log` 是隐藏文件，`*` 默认不含点开头的名字（见第 7 题）。

**2.** `?` = 恰好一个字符。
```bash
ls host?.conf       # → host1.conf  host2.conf
```
*为什么：* `host?.conf` 中 `?` 只顶一个字符 → `host1`/`host2` 命中；`host10`（两位）、`hostname`（多位）都不匹配。

**3.** `[…]` = 括号里任选一个字符。
```bash
ls host[12].conf    # → host1.conf  host2.conf
```
*为什么：* `[12]` 匹配 `1` 或 `2` 单个字符。等价写法 `host[1-2].conf`。

**4.** 范围 `[0-9]`。
```bash
ls app.log.[0-9]    # → app.log.1  app.log.2
```
*为什么：* `[0-9]` 匹配任一数字。常用还有 `[a-z]`、`[A-Z]`、`[a-zA-Z0-9]`。

**5.** 取反 `[!…]`（也可写 `[^…]`）。
```bash
ls [!a]*.log        # → error.log
```
*为什么：* `[!a]` = 首字符**不是** `a`。`app.log`、`access.log` 都以 `a` 开头被排除，只剩 `error.log`。

**6.** 未匹配时的默认行为 vs `nullglob`。
```bash
# 默认：模式匹配不到任何文件时，原样保留字符串
for f in *.xml; do echo "[$f]"; done
# → [*.xml]        （循环跑了 1 次，$f 就是字面 "*.xml"）

shopt -s nullglob
for f in *.xml; do echo "[$f]"; done
# → (什么都不打印，循环跑 0 次)
shopt -u nullglob   # 用完关掉，恢复默认
```
*为什么（重要坑）：* Bash **默认**在 glob 无匹配时把模式**当字面字符串**传下去——这会让 `for f in *.xml` 误把 `*.xml` 当成一个"文件名"处理，酿成 bug。运维脚本里遍历文件，建议开 `nullglob`（无匹配就跳过），或先判断文件是否存在。

**7.** 隐藏文件与 `dotglob`。
```bash
echo *              # 不含 .env / .hidden.log
shopt -s dotglob
echo *              # → .env .hidden.log access.log app.log ... （含隐藏文件）
shopt -u dotglob
```
*为什么：* `*` **默认不匹配以 `.` 开头**的名字（这就是隐藏文件的定义）。开 `dotglob` 后 `*` 连隐藏文件一起匹配（但仍不含 `.` 和 `..`）。备份/清理脚本要小心这条，否则会漏掉 `.env`、`.ssh` 之类。

**8.** 扩展通配 `extglob`。
```bash
shopt -s extglob                 # ← 必须先单独打开
ls *.@(csv|txt)                  # → data.csv  data.txt      @(a|b)=其中之一
echo !(*.log)                    # → 除 *.log 外的所有名字   !(pat)=非该模式
```
extglob 五个算子：`?(p)` 0或1次 · `*(p)` 0或多次 · `+(p)` 1或多次 · `@(p)` 恰好其一 · `!(p)` 取反。
*⚠️ 解析时机坑：* `extglob` 要在 shell **解析到**扩展模式**之前**就生效。
- ✅ 交互 shell：先敲 `shopt -s extglob`，回车，再用模式。
- ✅ 脚本：把 `shopt -s extglob` 放**第一行**（脚本逐行读，后面的模式就 OK）。
- ❌ `bash -c 'shopt -s extglob; ls *.@(csv|txt)'`：整串**一次性解析**，此刻 extglob 还没开 → `syntax error near unexpected token '('`。

---

## 一句话总结（B1.2 核心）
- **glob ≠ 正则**：`*`=任意串、`?`=单字符、`[abc]`/`[a-z]`=集合/范围、`[!abc]`=取反；且**只匹配已存在的文件名**。
- 三个 `shopt` 开关运维常踩：**`nullglob`**（无匹配→空，防"字面模式"bug）、**`dotglob`**（含隐藏文件）、**`extglob`**（`@()/!()/+()`…，注意开启时机）。
- 默认坑：无匹配保留字面、`*` 不含隐藏文件——写文件遍历脚本前先想清楚这两条。
