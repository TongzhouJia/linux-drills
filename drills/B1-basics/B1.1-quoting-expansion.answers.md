# KP B1.1 答案 — Quoting & Expansion Order

> 每题给「命令 + 输出 + 为什么」。命令都在真实 bash 里跑过、输出核对无误。你机器上 hostname/日期会不同，属正常。

---

**1.** 字面输出特殊字符 → **单引号**最省事（单引号内一切都不解释）。
```bash
echo 'Price is $5 (100% off!)'
```
输出：`Price is $5 (100% off!)`
*为什么：* 单引号 `'...'` 里 `$ ( ) %` 全部字面。用双引号则 `$5` 会被当变量（空），`(...)` 在 `echo` 里虽无害但换到别处会触发子shell。

**2.** 单引号 vs 双引号的核心区别。
```bash
name=RHCSA
echo 'single: $name'     # → single: $name   （$name 不展开）
echo "double: $name"     # → double: RHCSA   （$name 展开）
```
*为什么：* **单引号不做任何展开，双引号会做变量/命令/算术展开**（但不做 word-splitting 和 globbing）。这是 Bash 引用的第一原则。

**3.** 命令替换 `$( )` 存进变量。
```bash
h=$(hostname -s)
echo "Host is $h"        # → Host is server1
```
*为什么：* `$(cmd)` 把命令的**标准输出**替换进来。等号两边不能有空格。旧写法 `` `cmd` `` 也行，但 `$( )` 可嵌套、更清晰。

**4.** 算术展开 `$(( ))`。
```bash
echo $((24*60*60))       # → 86400
```
*为什么：* `$(( ))` 里是整数运算，可直接写 `*  / + - %`，变量还不用加 `$`（`$((a*b))` 也行）。注意 Bash 原生**不做浮点**，要小数得用 `bc`（后面 B3.8）。

**5.** 花括号展开建多文件（一条命令）。
```bash
touch report-{jan,feb,mar,apr}.log
```
生成：`report-jan.log report-feb.log report-mar.log report-apr.log`
*为什么：* `{a,b,c}` 是**逗号列表**展开，前后缀会分发到每一项：`report-` + 每项 + `.log`。

**6.** 花括号**序列**展开 + 补零。
```bash
mkdir node{01..10}
```
生成：`node01 node02 … node10`
*为什么：* `{01..10}` 是数字序列；**首项带前导 0（`01`）就自动补零对齐**。`{1..10}` 则是 `1 2 … 10` 不补零。

**7.** 波浪号展开取家目录。
```bash
echo ~root              # → /root
```
*为什么：* `~` = 当前用户家目录（`$HOME`），`~user` = 该用户家目录（查 `/etc/passwd`）。**注意 `~` 在引号内不展开**：`echo "~root"` 只会原样打印 `~root`。

**8.** 带空格的文件名——必须加引号。
```bash
touch "my report.txt"
cat "my report.txt"     # 成功读取
```
*为什么：* 不加引号时 `cat my report.txt` 会被拆成 `my` 和 `report.txt` **两个参数**，报两个 "No such file"。引号（或 `my\ report.txt` 转义空格）把它保持成一个参数。

**9.** 双引号里嵌命令替换。
```bash
echo "Today is $(date +%A)"     # → Today is Friday
```
*为什么：* **双引号内 `$( )` 照常展开**（双引号只挡 word-splitting/globbing，不挡变量/命令替换）。所以能把动态结果无缝拼进字符串。

**10.** 阻止通配符展开 → 引号（或转义）。
```bash
echo '*.txt'            # → *.txt
```
*为什么：* 不加引号时 `*.txt` 会被 shell 先展开成当前目录匹配的文件名（`a.txt b.txt c.txt`）再交给 `echo`。单引号让 `*` 保持字面。`"*.txt"` 或 `\*.txt` 同效。

**11.** `cp file{,.bak}` 备份惯用法（一条命令）。
```bash
cp config.ini{,.bak}   # 等价于 cp config.ini config.ini.bak
diff config.ini config.ini.bak && echo identical
```
*为什么：* `{,.bak}` 展开成**空串**和 `.bak` 两项，于是 `config.ini{,.bak}` → `config.ini config.ini.bak`。运维改配置前一键留底，非常常用。

**12.** 展开顺序综合题。
```bash
echo "{1..3} = $(echo {1..3})"     # → {1..3} = 1 2 3
```
*为什么（关键）：*
- **双引号内的 `{1..3}` 不展开** → 原样保留 `{1..3}`。
- 但 `$( )` 里的内容是**未加引号**的，其中的 `{1..3}` 正常花括号展开成 `1 2 3`，再由内层 `echo` 打印。
- 记忆点：**花括号展开发生在双引号之外**；变量/命令替换在双引号之内照做。这也是为什么 `echo "{a,b}"` 打印字面 `{a,b}`，而 `echo {a,b}` 打印 `a b`。

---

## 一句话总结（B1.1 核心）
- **单引号=全字面；双引号=展开变量/命令/算术，但不拆词不通配；不加引号=全都来（含通配、拆词）。**
- **花括号 `{}` 展开最早发生，且在双引号内失效**；波浪号 `~` 在引号内也失效。
- 运维默认习惯：**变量一律 `"$var"` 加双引号**，除非你明确需要拆词/通配。
