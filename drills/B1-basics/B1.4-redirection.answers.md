# KP B1.4 答案 — Redirection (advanced) / 重定向进阶

> 命令都在真实 bash 里跑过、输出核对无误。心法：**三根水管 0/1/2 = stdin/stdout/stderr**；`N>&M` 读作"把文件描述符 N 指向 M **当前**指的地方"。

---

**1.** `>` 截断，`>>` 追加。
```bash
echo "line1" > out.txt     # 新建/清空后写入
echo "line2" >> out.txt    # 追加
cat out.txt                # → line1 / line2
echo "second" > out.txt    # 再次 > : 文件被清空只剩这行
cat out.txt                # → second
```
*为什么：* `>` 每次都从头覆盖（文件先被截为 0 字节）。想保留历史一定用 `>>`。误用 `>` 覆盖日志是经典事故。

**2.** stdout 与 stderr 分流到两个文件。
```bash
ls /etc /nope > good.txt 2> err.txt
cat good.txt   # /etc 的列表
cat err.txt    # ls: cannot access '/nope': No such file or directory
```
*为什么：* `1>`（可简写 `>`）收 stdout，`2>` 收 stderr。两者是独立水管，可分别落盘。

**3.** 两条流合并进一个文件。
```bash
ls /etc /nope > all.txt 2>&1
grep -c 'No such' all.txt   # → 1，列表和错误都在里面
```
*为什么：* 先 `> all.txt` 把 stdout 指向文件，再 `2>&1` 把 stderr 也指向"stdout 现在指的地方"= 同一个文件。

**4.** ⭐ 顺序陷阱（这题最重要）。
```bash
# A（错）：错误没进文件，还打在屏幕上
ls /nope 2>&1 > t.txt
#   → 屏幕出现 "ls: cannot access '/nope'..."；t.txt 里没有错误

# B（对）：错误进了文件
ls /nope > t.txt 2>&1
#   → t.txt 里有错误
```
*为什么（逐步展开，重定向从左往右生效）：*
- **A**：先 `2>&1` —— 此刻 stdout 还指向**终端**，于是 stderr 也被指向**终端**；再 `> t.txt` 只把 stdout 改指文件。stderr 仍留在终端 → 错误没进文件。
- **B**：先 `> t.txt` —— stdout 指向文件；再 `2>&1` —— stderr 复制 stdout **当前**的指向（文件）→ 错误进文件。
- 记忆：**`2>&1` 要放在 `> file` 的后面**才能把错误也收进文件。`&>file` 是"两条都进 file"的简写，不用记顺序。

**5.** 丢弃全部输出，但保留退出码。
```bash
ls /etc /nope &>/dev/null
echo $?        # → 2   （输出没了，成败还能判断）
```
*为什么：* `&>/dev/null` = `>/dev/null 2>&1`，把 1 和 2 都倒进"黑洞"。常用于"我只关心命令成不成功，不想看输出"：`if command &>/dev/null; then ...`。

**6.** 只压制 stderr（去掉噪音，保留结果）。
```bash
find /etc -name '*.conf' 2>/dev/null | head -1
#   → /etc/sensors3.conf（示例）；没有一堆 "Permission denied"
```
*为什么：* `find` 遇到没权限的目录会往 stderr 刷 "Permission denied"。`2>/dev/null` 把这些噪音丢掉，真正的结果（stdout）照常输出。

**7.** here-doc（会展开）。
```bash
cat <<EOF
user=$USER date=$(date +%F)
EOF
#   → user=tongzhou date=2026-07-17   （变量、命令替换都展开）
```
*为什么：* `<<EOF` 把两行 `EOF` 之间的内容当 stdin 喂给命令；**未加引号的定界符**里，`$var`、`$( )`、算术都会展开。写配置文件、生成脚本常用。

**8.** here-doc（不展开）。
```bash
cat <<'EOF'
user=$USER date=$(date +%F)
EOF
#   → user=$USER date=$(date +%F)   （原样保留）
```
*为什么：* **给定界符加单引号 `<<'EOF'`**，块内一切字面输出。要往文件里写带 `$` 的模板（如别人的脚本、正则）时必须这么做，否则会被就地展开。
> 小技巧：`<<-EOF`（加连字符）允许用 **Tab** 缩进定界符和内容，便于在函数里对齐。

**9.** here-string `<<<`。
```bash
wc -w <<< "one two three"     # → 3
```
*为什么：* `<<<` 把**一个字符串**直接当 stdin，比 `echo "..." | cmd` 更省一个管道/进程。常见：`grep pat <<< "$line"`、`read a b c <<< "$line"`。

**10.** `tee`（分流）与 `exec`（整段重定向）。
```bash
echo hello | tee tlog.txt        # 屏幕显示 hello，同时写入 tlog.txt
echo world | tee -a tlog.txt     # -a 追加
cat tlog.txt                     # → hello / world

# 脚本开头一行，把之后所有输出（含错误）都送进日志：
( exec > run.log 2>&1
  echo "inside exec redirect"
  ls /nope )                     # 这两行的 stdout+stderr 都进 run.log
cat run.log
#   → inside exec redirect
#     ls: cannot access '/nope': No such file or directory
```
*为什么：* `tee` 让输出**既上屏又落盘**（还能同时喂给下一个管道），排障时特别顺手。`exec > file 2>&1` **改的是当前 shell 自己的 1/2**，此后整段脚本的输出统一进日志——定时任务/后台脚本记录日志的标准写法。

---

## 一句话总结（B1.4 核心）
- **1=stdout、2=stderr、0=stdin**；`>` 覆盖、`>>` 追加、`2>` 收错误。
- **合并错误要 `> file 2>&1`（2>&1 放后面）**；`&>file` 是两条都进的简写；`&>/dev/null` 丢弃一切但保留 `$?`。
- **here-doc**：`<<EOF` 展开、`<<'EOF'` 字面；**here-string** `<<<` 喂单个字符串。
- `tee` 上屏+落盘；`exec > log 2>&1` 把整段脚本输出接管进日志。
