# KP B1.3 — Parameter Expansion / 变量与参数展开
**`${v}` · `:-` `:=` `:?` · `${#v}` · `#/##` `%/%%` · `/` `//` · `:off:len` · `^^` `,,`**

**12 tasks · ⭐⭐⭐ · RHEL 10 · server1 as root · run each in a bash shell and match the stated result**

> 这一节是**运维日常最高频**的纯 Bash 功夫：切路径、取扩展名、抠日期、填默认值——很多时候根本不用 `sed`/`awk`，`${…}` 一把梭更快更稳。务必逐条真跑，看清 `#` vs `##`、`%` vs `%%`、`:-` vs `:=` 的差别。

---

**1.** With `base=report`, print `report_v2.log` using `${base}`. (Explain why `$base_v2` would fail.)
*Acceptance:* prints `report_v2.log`; uses `${base}` to keep the var name from swallowing `_v2`.

**2.** With `LOGDIR` **unset**, print `/var/log` as a fallback using `${var:-default}` — **without** assigning it. Then confirm `LOGDIR` is still unset.
*Acceptance:* prints `/var/log`; `LOGDIR` remains unset afterward.

**3.** With `PORT` **unset**, use `${var:=default}` to print `8080` **and** assign it, so `echo $PORT` afterward shows `8080`.
*Acceptance:* prints `8080`; `PORT` now equals `8080`.

**4.** Write a one-liner that aborts with a message if `USERNAME` is unset — the required-variable guard `${var:?message}`.
*Acceptance:* when `USERNAME` is unset it prints `bash: USERNAME: <message>` to **stderr** and returns non-zero.

**5.** With `pw='S3cr3t!'`, print its length using `${#var}`.
*Acceptance:* prints `7`.

**6.** With `f=archive.tar.gz`, get the extension `gz` using `${f##*.}`. Also show what `${f#*.}` gives, and explain `#` vs `##`.
*Acceptance:* `${f##*.}` → `gz`; `${f#*.}` → `tar.gz`.

**7.** With `f=report.2026.csv`, strip the extension using `${f%.*}` → `report.2026`, and `${f%%.*}` → `report`. Explain `%` vs `%%`.
*Acceptance:* `report.2026` and `report`.

**8.** With `p=/var/log/nginx/access.log`, get the basename `access.log` via `${p##*/}` and the dirname `/var/log/nginx` via `${p%/*}` — pure expansion, no `basename`/`dirname` commands.
*Acceptance:* `access.log` and `/var/log/nginx`.

**9.** With `s=a-b-c-d`, replace the **first** `-` with `_` (`${s/-/_}`) and **all** `-` with `_` (`${s//-/_}`).
*Acceptance:* `a_b-c-d` and `a_b_c_d`.

**10.** With `d=2026-07-17`, use substring expansion `${var:offset:length}` to extract the year `2026` and the month `07`.
*Acceptance:* `2026` (`${d:0:4}`) and `07` (`${d:5:2}`).

**11.** Case conversion: `name=server1` → uppercase `SERVER1` (`${name^^}`); `H=WEB-PROD` → lowercase `web-prod` (`${H,,}`).
*Acceptance:* `SERVER1` and `web-prod`.

**12.** Real chain — given `f=/data/backups/db-2026-07-17.sql.gz`, using **only parameter expansion** produce: (a) the bare filename, (b) the filename without `.sql.gz`, (c) the date `2026-07-17`.
*Acceptance:* (a) `db-2026-07-17.sql.gz` · (b) `db-2026-07-17` · (c) `2026-07-17`.
