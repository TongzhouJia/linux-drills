# KP B4.6 — Idempotent & Defensive Scripting
**幂等（可重复跑）· 先检查再动手 · `--dry-run` · `flock` 防并发 · 改前备份 · `--force` 护栏**

**8 tasks · ⭐⭐ · RHEL 10 · server1 · run each and match the result**

> "幂等" = **同一个脚本跑一次和跑十次，结果一样、不出错、不重复。** 这是运维脚本能不能挂进 cron/自动化的关键——不然重跑就出乱子。

---

**1.** Show `mkdir -p` is idempotent (run it twice, both succeed).
*Acceptance:* both runs exit `0`, no "already exists" error.

**2.** Append `PARAM=1` to a config **only if not already present** (run twice → still one line).
*Acceptance:* `grep -qxF "PARAM=1" conf || echo "PARAM=1" >> conf` run twice → `conf` has 1 line.

**3.** Add a `--dry-run` flag that prints actions instead of doing them.
*Acceptance:* with `--dry-run` the script prints `DRY: <cmd>`; without it, it runs.

**4.** Use `flock` so a second concurrent run is **locked out** instead of overlapping.
*Acceptance:* first run holds the lock; a second run while it's held prints `LOCKED OUT` and exits.

**5.** **Back up** a config file before editing it.
*Acceptance:* `cp -a app.conf app.conf.bak.$(date +%F)` → a dated backup exists.

**6.** Guard a destructive action behind `--force`: refuse (exit 1) unless `--force` is passed.
*Acceptance:* no flag → `refusing (need --force)`, non-zero; `--force` → `proceeding`.

**7.** Make a symlink point somewhere **idempotently** with `ln -sfn` (run twice, last wins, no error).
*Acceptance:* `ln -sfn tgt link` twice → `link` points to the target, no "File exists" error.

**8.** Write an `ensure_line <line> <file>` function that adds a line only if absent (run twice → one line).
*Acceptance:* `ensure_line "10.0.0.1 web1" hosts.txt` twice → `hosts.txt` has 1 line.
