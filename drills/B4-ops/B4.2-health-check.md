# KP B4.2 — System Health Check ⭐⭐⭐
**解析 `df`/`free`/`ps`/`uptime` 出阈值告警：磁盘、内存、CPU 负载、进程**

**12 tasks · ⭐⭐⭐ · RHEL 10 · server1 · run each (values depend on the machine — match the shape/logic)**

> 这些命令在真机上跑，**输出数字随机器不同**（磁盘/内存/负载都会变），所以验收看的是**命令写法和逻辑对不对**，不是具体数值。健康检查是运维脚本第二大用途（第一是日志分析）。

---

**1.** Print disk usage % for each mounted filesystem (parse `df -P`).
*Acceptance:* `df -P | awk 'NR>1{u=$5; gsub(/%/,"",u); print $6, u"%"}'` → `/ 55%`, … (one line per mount).

**2.** **Alert** on any filesystem over a threshold (e.g. 80%).
*Acceptance:* `df -P | awk 'NR>1{u=$5; gsub(/%/,"",u); if(u+0>80) print "ALERT", $6, $5}'` → lines only for over-threshold mounts.

**3.** Just the **root** filesystem's usage %.
*Acceptance:* `df -P / | awk 'NR==2{print $5}'` → e.g. `55%`.

**4.** Memory **used %** (parse `free`).
*Acceptance:* `free | awk '/^Mem:/{printf "%.1f%%\n", 100*$3/$2}'` → e.g. `5.0%`.

**5.** **Available** memory in MB.
*Acceptance:* `free -m | awk '/^Mem:/{print $7" MB"}'` → e.g. `3554 MB`.

**6.** 1-minute **load average** vs number of cores.
*Acceptance:* `load=$(uptime | sed -E 's/.*load average: *//' | cut -d, -f1); echo "$load / $(nproc) cores"`.

**7.** Top 3 processes by **memory**.
*Acceptance:* `ps -eo pid,comm,%mem --sort=-%mem | head -4` → header + 3 hungriest.

**8.** Top 3 processes by **CPU**.
*Acceptance:* `ps -eo pid,comm,%cpu --sort=-%cpu | head -4`.

**9.** **Swap** used / total.
*Acceptance:* `free -m | awk '/^Swap:/{print $3"/"$2" MB used"}'`.

**10.** Total number of running processes.
*Acceptance:* `ps -e --no-headers | wc -l` → a number.

**11.** Write `healthcheck.sh [disk_max] [mem_max]` that checks the top filesystems and memory against thresholds, prints `OK`/`WARN` per item, and **exits non-zero if anything is WARN** (so cron/monitoring can act on it).
*Acceptance:* with a low disk threshold → prints `WARN disk … NN%` lines and exits `1`; all-clear → all `OK`, exit `0`.

**12.** Disk usage of a specific directory (not filesystem) with `du`.
*Acceptance:* `du -sh /var/log` → a human-readable size like `12M	/var/log`.
