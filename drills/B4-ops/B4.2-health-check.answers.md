# KP B4.2 答案 — System Health Check

> 命令都在真实 bash 里跑过（数值随机器变，写法/逻辑核对无误）。心法：**系统命令的输出用 `awk` 抠数字 → 和阈值比 → 超了就告警 + 非 0 退出。**

---

**1.** 各挂载点磁盘 %。
```bash
df -P | awk 'NR>1{u=$5; gsub(/%/,"",u); print $6, u"%"}'
# → / 55%  /run 1%  ...
```
*为什么：* `df -P` 输出稳定（POSIX 格式，一行一个挂载点）。第 5 列是 `55%`，`gsub(/%/,"",u)` 去掉百分号变成数字好比较，第 6 列是挂载点。

**2.** 超阈值告警。
```bash
df -P | awk 'NR>1{u=$5; gsub(/%/,"",u); if(u+0>80) print "ALERT", $6, $5}'
```
*为什么：* `u+0` 强制转成数字再比。只打印超过 80% 的行——这就是"磁盘要满了"告警的核心。

**3.** 根分区。
```bash
df -P / | awk 'NR==2{print $5}'      # → 55%
```

**4–5. 内存。**
```bash
free | awk '/^Mem:/{printf "%.1f%%\n", 100*$3/$2}'   # 已用% ：used/total
free -m | awk '/^Mem:/{print $7" MB"}'               # available（第7列）
```
*为什么：* `free` 的 `Mem:` 行：$2=total、$3=used、$7=available。**判断内存紧张看 `available` 比 `free` 准**（free 不含可回收的 buff/cache）。

**6.** 负载 vs 核数。
```bash
load=$(uptime | sed -E 's/.*load average: *//' | cut -d, -f1)
echo "$load / $(nproc) cores"
```
*为什么：* `uptime` 末尾是 `load average: 0.00, 0.05, 0.10`（1/5/15 分钟）。`sed` 砍到冒号后、`cut` 取第一个。**判断 CPU 是否过载：1 分钟负载 > 核数就是满了**（`nproc` 给核数）。

**7–8. 最吃资源的进程。**
```bash
ps -eo pid,comm,%mem --sort=-%mem | head -4    # 内存 top3
ps -eo pid,comm,%cpu --sort=-%cpu | head -4    # CPU top3
```
*为什么：* `-eo` 自选列、`--sort=-%mem` 按内存降序（负号=降序）。`head -4` = 表头 + 3 行。定位"谁在吃内存/CPU"，配 B4.3 去处理。

**9.** Swap。
```bash
free -m | awk '/^Swap:/{print $3"/"$2" MB used"}'
```
*为什么：* Swap 用得多通常意味着内存不够、性能下降，值得监控。

**10.** 进程总数。
```bash
ps -e --no-headers | wc -l
```

**11.** 健康检查脚本（带阈值 + 退出码）。
```bash
#!/usr/bin/env bash
set -euo pipefail
DISK_MAX=${1:-80}; MEM_MAX=${2:-90}
status=0
while read -r use mnt; do
  if (( use > DISK_MAX )); then echo "WARN disk $mnt ${use}%"; status=1
  else echo "OK   disk $mnt ${use}%"; fi
done < <(df -P | awk 'NR>1{u=$5; gsub(/%/,"",u); print u, $6}' | sort -rn | head -3)
memuse=$(free | awk '/^Mem:/{printf "%d", 100*$3/$2}')
if (( memuse > MEM_MAX )); then echo "WARN mem ${memuse}%"; status=1; else echo "OK   mem ${memuse}%"; fi
exit $status
```
运行 `healthcheck.sh 50 90`（磁盘阈值 50%）→ 超标的挂载点打 `WARN …`、内存 `OK 5%`、**退出码 1**。
*为什么：* **关键是 `exit $status`**——脚本用退出码告诉调用方"有没有告警"。挂到 cron/systemd timer（B5.3），或让监控系统按退出码触发通知。阈值做成参数，不同机器复用同一个脚本。
> 注意：读 df 用了 `done < <(…)` 进程替换而非 `df | while`，否则 `status` 在子shell里改了会丢（B3.3 的坑）。

**12.** 目录占用 `du`。
```bash
du -sh /var/log        # → 例如 12M  /var/log
```
*为什么：* `df` 看**文件系统**整体，`du` 看**某个目录**占多少（`-s` 汇总、`-h` 人类可读）。找"谁占了空间"：`du -sh /var/* | sort -rh | head`。

---

## 一句话总结（B4.2 核心）
- **磁盘** `df -P`+awk 去 `%` 比阈值；**目录** `du -sh`；**内存** `free` 看 `available`；**负载** `uptime` 末尾 vs `nproc`。
- **最吃资源进程** `ps -eo … --sort=-%mem|%cpu | head`。
- **健康检查脚本**：阈值做参数、逐项 `OK/WARN`、**`exit $status`** 让 cron/监控能据此告警；读 df 用 `< <(…)` 防子shell丢变量。
