# KP B4.4 答案 — Batch Users / Files / Permissions

> 文件/权限题全真机跑过、输出核对无误；建用户用 `--dry-run` 验证逻辑（真建需 root）。心法：**CSV → `while IFS=, read` 逐行；批量改权限 → `find … -exec … +`；建东西前先判存在（幂等）。**

---

**1.** CSV 生成 useradd（dry-run）。
```bash
while IFS=, read -r u full grp sh; do
  echo "useradd -c \"$full\" -G $grp -s $sh $u"
done < <(tail -n +2 users.csv)
```
*为什么：* `tail -n +2` 跳表头；`IFS=,` 按逗号拆字段。先 `echo` 出命令肉眼核对，确认无误再去掉 echo 真跑。

**2 / 10. 幂等批量建用户脚本。**
```bash
#!/usr/bin/env bash
set -euo pipefail
DRY=0; [[ "${1:-}" == "--dry-run" ]] && { DRY=1; shift; }
csv=${1:?usage: mkusers.sh [--dry-run] <csv>}
while IFS=, read -r u full grp sh; do
  if id "$u" &>/dev/null; then echo "skip $u (exists)"; continue; fi   # 幂等
  cmd=(useradd -c "$full" -s "$sh" "$u")
  if (( DRY )); then echo "DRY: ${cmd[*]}"; else "${cmd[@]}" && echo "created $u"; fi
done < <(tail -n +2 "$csv")
```
*为什么：* **`id "$u" &>/dev/null` 判断用户是否已存在**——存在就跳过，脚本可**反复安全运行**（幂等，B4.6 主题）。`--dry-run` 让你先看命令不动真格。命令存进**数组** `cmd=(...)` 再执行，能正确处理带空格的 `"$full"`。

**3.** 批量建目录树。
```bash
mkdir -p proj/{src,bin,doc}
touch proj/src/file{1..3}.txt
```
*为什么：* `mkdir -p` 一次建多层且已存在不报错（本身幂等）；花括号展开一把建多个。

**4.** 查危险权限并修。
```bash
find proj -type f -perm -0002 -exec chmod o-w {} +    # 777 → 775
```
*为什么：* `-perm -0002` = "**其他人可写**"（`-` 前缀表示"包含这些位"）。`-exec … {} +` 把匹配的文件**批量**喂给一条 `chmod`（比 `\;` 一个个调高效）。

**5.** 按属主找文件。
```bash
find proj -type f -user "$(whoami)" | wc -l
```
*为什么：* `-user` 按属主过滤。审计"谁的文件"、清理离职员工遗留文件常用（`-user olduser`）。

**6.** 递归规范权限（文件/目录分开）。
```bash
find proj -type f -exec chmod 644 {} +     # 文件 644
find proj -type d -exec chmod 755 {} +     # 目录 755
```
*为什么：* **文件和目录权限要分开设**——目录需要 `x`（进入权）所以 755，文件不需要所以 644。千万别 `chmod -R 755` 一刀切（会给所有文件加执行位）。

**7–8. 权限审计。**
```bash
find proj -perm -0002                       # world-writable（谁都能改，风险）
find /usr/bin -perm -4000 -type f           # SUID（以属主身份运行，重点审计对象）
```
*为什么：* 安全巡检两大项：**world-writable**（任何人可改，可能被植入）和 **SUID**（`-4000`，运行时提权到属主，被利用就是提权漏洞）。`/usr/bin/passwd` 正常有 SUID（改密码要写 /etc/shadow）；出现不该有的 SUID 就要警惕。

**9.** 批量重命名。
```bash
cd proj/src
for f in *.txt; do mv "$f" "2026_$f"; done   # → 2026_file1.txt ...
```
*为什么：* 循环 + `mv` 加前缀。注意 `"$f"` 加引号防空格文件名。改扩展名：`mv "$f" "${f%.txt}.log"`（B1.3 参数展开）。

---

## 一句话总结（B4.4 核心）
- **CSV 驱动**：`while IFS=, read -r … < <(tail -n +2 file)` 逐行；命令存数组处理空格。
- **幂等**：建之前先判存在（`id user`、`[[ -e ]]`、`mkdir -p`），脚本可反复跑。
- **批量改权限**：`find … -type f/d -perm … -exec chmod … {} +`；文件 644 / 目录 755 分开设，别 `-R` 一刀切。
- **审计**：world-writable `-perm -0002`、SUID `-perm -4000`；按属主 `-user`。
