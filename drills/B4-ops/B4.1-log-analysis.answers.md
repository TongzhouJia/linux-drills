# KP B4.1 答案 — Log Analysis

> 全部在真实 bash + 样例日志上跑过、输出核对无误。心法：**日志分析 = `grep 筛` + `awk 取/算` + `sort|uniq -c|sort -rn|head` 排名**，反复就这套。

---

**1.** 总量与独立访客。
```bash
echo "total=$(wc -l < access.log) unique=$(awk '{print $1}' access.log | sort -u | wc -l)"
# → total=30 unique=6
```

**2.** top 5 IP。
```bash
awk '{print $1}' access.log | sort | uniq -c | sort -rn | head -5
# → 10.0.0.1(10) 10.0.0.2(5) 203.0.113.7(4) 192.168.1.50(4) 10.0.0.3(4)
```

**3.** 各状态码计数。
```bash
awk '{print $9}' access.log | sort | uniq -c | sort -rn
# → 200×17 · 404×5 · 500×3 · 301×2 · 503×2 · 401×1
```

**4.** 错误率（≥400 占比）。
```bash
awk '$9>=400{e++} END{printf "%.1f%% (%d/%d)\n", 100*e/NR, e, NR}' access.log
# → 36.7% (11/30)
```
*为什么：* 一遍扫描同时累加错误数 `e` 和总数 `NR`，`END` 里算百分比。SLA/健康度的核心指标。

**5.** ⭐ 谁在制造 5xx（最实战）。
```bash
awk '$9 ~ /^5/{print $1}' access.log | sort | uniq -c | sort -rn | head -3
# → 4 10.0.0.1 · 1 203.0.113.7
```
*为什么：* `$9 ~ /^5/` 筛 5xx（500/503），按来源 IP 排名。线上报错时第一时间锁定"谁在打挂服务"。这正是蓝图里那道"扫日志找造成 5xx 最多的 IP"的核心。

**6.** 总流量与平均。
```bash
awk '{s+=$NF; n++} END{printf "total=%d avg=%.1f\n", s, s/n}' access.log
# → total=23936 avg=797.9
```

**7.** 按小时分布。
```bash
awk '{split($4,a,":"); c[a[2]]++} END{for(h in c) print h, c[h]}' access.log | sort
# → 10 14 / 11 16
```
*为什么：* `split($4,a,":")` 从 `[17/Jul/2026:10:00:01` 抠出小时 `a[2]`。看流量趋势、定位高峰。

**8.** 最热 URL。
```bash
awk '{print $7}' access.log | sort | uniq -c | sort -rn | head -1
# → 7 /index.html
```

**9.** 失败登录来源（安全）。
```bash
awk '/Failed password/{for(i=1;i<=NF;i++) if($i=="from") c[$(i+1)]++} END{for(x in c) print c[x],x}' syslog.log | sort -rn
# → 3 203.0.113.7 · 1 198.51.100.9
```
*为什么：* 定位 SSH 爆破源，配合 firewalld/fail2ban 封 IP。

**10.** 时间窗过滤。
```bash
awk '$4 ~ /:11:0[0-5]:/' access.log | wc -l      # → 6
```
*为什么：* 正则 `:11:0[0-5]:` 匹配 11:00–11:05 的时间戳。"只看某段时间的日志"极常用（配合告警"最近 5 分钟错误数"）。

**11.** 最忙的小时。
```bash
awk '{split($4,a,":"); c[a[2]]++} END{for(h in c) print c[h],h}' access.log | sort -rn | head -1
# → 16 11
```

**12.** 汇总报表脚本。
```bash
#!/usr/bin/env bash
set -euo pipefail
log=${1:?usage: weblog-report.sh <logfile>}
echo "== Web log report: $log =="
printf "%-16s %s\n" "Total requests:" "$(wc -l < "$log")"
printf "%-16s %s\n" "Unique IPs:"     "$(awk '{print $1}' "$log" | sort -u | wc -l)"
printf "%-16s %s\n" "Top IP:"         "$(awk '{print $1}' "$log" | sort | uniq -c | sort -rn | head -1 | awk '{print $2" ("$1")"}')"
printf "%-16s %s\n" "Error requests:" "$(awk '$9>=400' "$log" | wc -l)"
printf "%-16s %s\n" "Top status:"     "$(awk '{print $9}' "$log" | sort | uniq -c | sort -rn | head -1 | awk '{print $2" ("$1")"}')"
```
输出（对 `access.log`）：
```
== Web log report: access.log ==
Total requests:  30
Unique IPs:      6
Top IP:          10.0.0.1 (10)
Error requests:  11
Top status:      200 (17)
```
*为什么：* 把常用统计封成一个带参脚本（`${1:?}` 强制传日志名）。挂到 cron 每天发一份，就是最朴素的"日志日报"。

---

## 一句话总结（B4.1 核心）
- **通用套路**：`grep/awk 筛+取` → `sort | uniq -c | sort -rn | head` 排名。
- **关键指标**：总量/独立 IP、状态分布、**错误率**、**5xx 来源 top-N**、按小时趋势、最热 URL、失败登录源。
- 时间窗用正则匹配时间戳字段；最后把这些拼成一个 `report.sh` 挂 cron 出日报。
