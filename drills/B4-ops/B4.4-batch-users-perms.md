# KP B4.4 — Batch Users / Files / Permissions
**CSV 驱动批量建用户 · `find -exec` 批量改权限 · 权限审计（world-writable / SUID）· 批量重命名 · 幂等**

**10 tasks · ⭐⭐ · RHEL 10 · server1 · 建用户需 root/sudo；文件/权限题任意环境可跑**

> 样例 `samples/users.csv`（`username,fullname,group,shell`）。运维常要"照一张清单批量建账号、统一改一片文件权限、审计有没有危险权限"。⚠️ `useradd` 要 root——练习先用 `--dry-run`/`echo` 看命令对不对，再真跑。

---

**1.** Read `users.csv` (skip header) and **print** the `useradd` command for each user (dry-run).
*Acceptance:* `while IFS=, read -r u full grp sh; do echo "useradd -c \"$full\" -G $grp -s $sh $u"; done < <(tail -n +2 users.csv)` → one command per user.

**2.** Same, but **actually create** the users (needs root). Make it **idempotent**: skip users that already exist (`id "$u"`).
*Acceptance:* existing users → `skip <u> (exists)`; new ones → created (or `DRY:` line with `--dry-run`).

**3.** Batch-create a directory tree `proj/{src,bin,doc}` and three files under `src`.
*Acceptance:* `mkdir -p proj/{src,bin,doc}; touch proj/src/file{1..3}.txt` → the 3 files exist.

**4.** Find world-writable files under `proj` and remove the world-write bit.
*Acceptance:* a `777` file → after `find proj -type f -perm -0002 -exec chmod o-w {} +` becomes `775`.

**5.** Count files under `proj` owned by you.
*Acceptance:* `find proj -type f -user "$(whoami)" | wc -l`.

**6.** Recursively normalize permissions: files → `644`, directories → `755`.
*Acceptance:* `find proj -type f -exec chmod 644 {} +` and `-type d -exec chmod 755 {} +` → file `644`, dir `755`.

**7.** **Audit**: list every world-writable path under `proj`.
*Acceptance:* `find proj -perm -0002` → the world-writable entries.

**8.** **Audit**: list SUID binaries in `/usr/bin` (security review).
*Acceptance:* `find /usr/bin -perm -4000 -type f` → e.g. `/usr/bin/passwd`, `/usr/bin/mount`, …

**9.** Bulk-rename all `*.txt` in `proj/src` to add a `2026_` prefix.
*Acceptance:* `for f in *.txt; do mv "$f" "2026_$f"; done` → `2026_file1.txt`, …

**10.** Write `mkusers.sh [--dry-run] <csv>` that creates users from a CSV, skips existing ones, and supports a dry-run flag.
*Acceptance:* `./mkusers.sh --dry-run users.csv` → `DRY: useradd … jsmith`, … (and `skip` for any that exist).
