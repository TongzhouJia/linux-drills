# KP B4.3 答案 — Process & Service Monitoring

> 全部真机跑过、输出核对无误（`kill`/`pkill` 用自起的 `sleep` 当靶子）。心法：**`pgrep` 找、`pkill` 杀、`systemctl is-active --quiet` 判活、退出码驱动动作。**

---

**准备靶子：** `sleep 300 & ; sleep 300 &`（起两个假进程）。

**1.** 是否在跑。
```bash
pgrep -x sleep >/dev/null && echo yes || echo no      # → yes
```
*为什么：* `pgrep -x name` 按**精确名**查进程，返回 0=找到。比 `ps aux | grep name`（会匹配到 grep 自己）干净得多。

**2–3. PID 与计数。**
```bash
pgrep -x sleep          # 每个 PID 一行
pgrep -xc sleep         # 计数（-c）
```

**4–5. 服务状态。**
```bash
systemctl is-active cron     # → active   (RHEL: crond)
systemctl is-enabled cron    # → enabled
```
*为什么：* `is-active` 看**当前**是否运行、`is-enabled` 看**开机**是否自启，两回事。巡检时两个都要查（跑着但没设自启，重启就没了）。

**6.** Watchdog（挂了就拉起）。
```bash
svc=cron
if systemctl is-active --quiet "$svc"; then
  echo "$svc up"
else
  echo "$svc DOWN"; systemctl restart "$svc"    # 真环境里拉起
fi
```
*为什么：* `--quiet` 不打印、只用退出码，适合放 `if`。挂 cron 每分钟跑一次 = 最简易的服务守护。

**7.** 最占 CPU 的进程。
```bash
ps -eo pid,comm,%cpu --sort=-%cpu --no-headers | head -1
```
*为什么：* 找到"跑飞的进程"的 PID，接下来可 `renice` 降优先级（RHCSA D3.4）或 `kill`。

**8.** 按 PID 精确杀。
```bash
kill "$PID"            # 默认发 SIGTERM（优雅退出）
# 不理会再： kill -9 "$PID"   （SIGKILL 强杀，最后手段）
```
*为什么：* 先 `kill`（TERM，让进程善后），无效再 `-9`（KILL，内核直接干掉、进程没机会清理）。别上来就 `-9`。

**9.** 按名字批量杀。
```bash
pkill -x sleep         # 杀掉所有精确名为 sleep 的进程
```
*为什么：* `pkill` = `pgrep` 的杀手版，按名/模式杀一批。⚠️ 模式别写太宽（`pkill ssh` 可能连你的 SSH 会话一起杀），能精确就 `-x`。

**10.** 服务巡检脚本。
```bash
#!/usr/bin/env bash
set -uo pipefail
status=0
for svc in "$@"; do
  if systemctl is-active --quiet "$svc"; then
    echo "OK   $svc active"
  else
    echo "DOWN $svc  (state: $(systemctl is-active "$svc"))"
    status=1
  fi
done
exit $status
```
运行 `svcmon.sh cron nosuchsvc` → `OK cron active` / `DOWN nosuchsvc (state: inactive)` / 退出码 1。
*为什么：* 遍历一串服务名逐个查，有一个挂就 `status=1`、最后 `exit $status`。挂到 timer 定期跑，退出非 0 时告警。注意这里没用 `set -e`（`is-active` 对 down 的服务返回非 0 是**预期**的，不该让脚本中止）——**该失败的地方别开 `-e`**。

---

## 一句话总结（B4.3 核心）
- **进程**：`pgrep -x` 找 / `-c` 计数、`pkill -x` 杀（精确名，别误伤）；先 `kill`(TERM) 再 `-9`(KILL)。
- **服务**：`is-active`（在跑吗）+ `is-enabled`（开机自启吗），`--quiet` 配 `if` 做 watchdog。
- **巡检脚本**遍历服务列表、有 down 就 `exit 1`；`is-active` 会正常返回非 0，这种脚本别开 `set -e`。
- RHEL 服务名：`crond`、`sshd`、`firewalld`……
