# KP B4.6 答案 — Idempotent & Defensive Scripting

> 全部真机跑过、输出核对无误。心法：**动手前先问三句——"已经是目标状态了吗？会不会和别的实例撞车？出错了能回退吗？"**

---

**1.** `mkdir -p` 天生幂等。
```bash
mkdir -p x/y; mkdir -p x/y     # 两次都 exit 0，已存在不报错
```
*为什么：* `-p` 存在即跳过、不存在才建。很多命令有幂等选项（`ln -sf`、`rm -f`、`useradd` 之前先 `id`）。优先用它们。

**2.** 存在才追加（配置去重）。
```bash
grep -qxF "PARAM=1" conf || echo "PARAM=1" >> conf
```
*为什么：* `grep -q`(静默) `-x`(整行匹配) `-F`(字面不当正则)——**先查有没有，没有才追加**。否则脚本每跑一次配置就多一行重复。改配置文件的幂等标准写法。

**3.** `--dry-run` 演习模式。
```bash
DRY=0; [[ "${1:-}" == "--dry-run" ]] && { DRY=1; shift; }
run(){ if (( DRY )); then echo "DRY: $*"; else "$@"; fi; }
run rm -f /tmp/old.log        # 带 --dry-run 只打印，不真删
```
*为什么：* 危险操作先 `--dry-run` 看"会做什么"，确认无误再真跑。封个 `run` 包装函数，全脚本统一走它。

**4.** `flock` 防并发。
```bash
( flock -n 9 || { echo "LOCKED OUT"; exit 1; }
  echo "holding lock"; sleep 1
) 9>/tmp/my.lock
```
*为什么：* `flock -n 9` 对文件描述符 9 加非阻塞锁，抢不到就 `||` 退出。**第二个实例在第一个跑完前会被挡住**——防止 cron 上一轮没跑完又启一轮、两个备份互相踩。比 B3.7 的 `mkdir` 锁更标准（进程结束自动释放，不会留死锁）。

**5.** 改前备份。
```bash
cp -a app.conf "app.conf.bak.$(date +%F)"
# 然后再 sed -i / 编辑 app.conf
```
*为什么：* **动配置前先留一份**，改错了能秒回退。`-a` 保留权限属性；带日期避免覆盖旧备份。配合 `sed -i.bak`（B2.2）双保险。

**6.** `--force` 护栏。
```bash
[[ "${1:-}" == "--force" ]] || { echo "refusing (need --force)"; exit 1; }
echo "proceeding"   # 危险操作放这后面
```
*为什么：* 删数据、重置这类**不可逆**操作，默认拒绝、必须显式 `--force` 才干。防手滑。可再加交互确认 `read -p "type YES: " a; [[ $a == YES ]] || exit`。

**7.** 幂等符号链接。
```bash
ln -sfn tgt link      # -s 软链 -f 强制覆盖 -n 不跟随已存在的链接目录
```
*为什么：* 直接 `ln -s` 遇已存在会报 "File exists"。`-sfn` 让它**幂等**：无论跑几次，`link` 都指向 `tgt`，不报错。切换"当前版本"软链（`current -> release-N`）常用。

**8.** `ensure_line` 幂等函数。
```bash
ensure_line(){ local line=$1 file=$2; grep -qxF "$line" "$file" 2>/dev/null || echo "$line" >> "$file"; }
ensure_line "10.0.0.1 web1" /etc/hosts    # 跑几次都只有一行
```
*为什么：* 把第 2 题封成可复用函数——"确保某行在文件里"。改 `/etc/hosts`、`authorized_keys`、配置片段都用得上，重复跑安全。

---

## 一句话总结（B4.6 核心）
- **幂等三招**：用幂等命令（`mkdir -p`/`ln -sfn`）、**先检查再动手**（`grep -q || …`、`id user`、`[[ -e ]]`）、封 `ensure_*` 函数。
- **防御**：`--dry-run` 先演习、`--force` 守护不可逆操作、改前 `cp .bak`、`flock` 防并发。
- 目标：**脚本重复跑不出乱子**，才能放心挂进自动化。
