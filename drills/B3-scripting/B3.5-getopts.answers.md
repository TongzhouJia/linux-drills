# KP B3.5 答案 — Option Parsing with `getopts`

> 参考脚本的所有调用都真机跑过、输出核对无误。心法：**optstring 里选项后加 `:` = 需要参数（值进 `$OPTARG`）；开头加 `:` = 静默模式，缺参/未知选项由你自己在 `case` 里处理。**

---

### 各调用与输出

| 调用 | 输出 | 退出码 |
|------|------|--------|
| `-v -n Alice -f data.txt extra1 extra2` | `verbose=1 name=Alice file=data.txt rest=extra1 extra2` | 0 |
| `-f only.txt` | `verbose=0 name=guest file=only.txt rest=` | 0 |
| `-f`（缺参数） | `Option -f requires an argument`（stderr） | 1 |
| `-x`（未知选项） | `Invalid option: -x` + usage（stderr） | 1 |
| `-v`（缺必填 -f） | `Error: -f is required`（stderr） | 1 |
| `-vf data.txt`（合并） | `verbose=1 name=guest file=data.txt rest=` | 0 |

---

### 逐条讲解

**1–3. 标志与带参选项。**
```bash
while getopts ":vn:f:" opt; do
  case $opt in
    v) verbose=1 ;;
    n) name=$OPTARG ;;
    f) file=$OPTARG ;;
    ...
```
- optstring `":vn:f:"`：`v` 无冒号=**布尔标志**；`n:` `f:` 有冒号=**需要参数**，参数值自动放进 `$OPTARG`。
- `getopts` 每次循环解析一个选项，`$opt` 是选项字母。

**4. 默认值。** 解析前先设 `verbose=0 name=guest file=""`，没传的选项就保持默认。这是给可选参数兜底的标准做法。

**5. 缺参数（`:` 分支）。**
```bash
:) echo "Option -$OPTARG requires an argument" >&2; exit 1 ;;
```
- optstring **开头的 `:`** 打开"静默错误模式"：当某选项缺参数时，`getopts` 把 `$opt` 设为 `:`、`$OPTARG` 设为出问题的选项字母，交给你处理（而不是它自己乱打印）。

**6. 未知选项（`\?` 分支）。**
```bash
\?) echo "Invalid option: -$OPTARG" >&2; usage ;;
```
- 传了没定义的选项（如 `-x`），`$opt` = `?`，`$OPTARG` = `x`。case 里写 `\?`（转义，避免被当通配）。

**7. 必填检查。**
```bash
[[ -n $file ]] || { echo "Error: -f is required" >&2; exit 1; }
```
- `getopts` 只解析，**"必填"要自己在循环后检查**。`-f` 没给就报错退出。

**8. 合并标志 + `shift`。**
```bash
shift $((OPTIND-1))
```
- `-vf data.txt` 等价 `-v -f data.txt`（短标志可合并）。
- `OPTIND` 是"下一个待处理参数的下标"；`shift $((OPTIND-1))` 把已解析的选项移走，`$1 $2 …`/`$*` 就只剩**位置参数**（这里是 `rest`）。

> 💡 额外一课：usage 和错误信息都用 `>&2` 打到 **stderr**。正常输出走 stdout、诊断信息走 stderr，这样别人 `脚本 > out.txt` 时错误仍看得见，管道里也不会污染数据。（stderr 无缓冲、stdout 有缓冲，终端里两者顺序偶尔看着错位，属正常。）

---

## 一句话总结（B3.5 核心）
- `while getopts ":vn:f:" opt; do case $opt in …; esac; done`：**选项后 `:` = 带参（值在 `$OPTARG`）**，**串首 `:` = 自己处理错误**（`:` 缺参 / `\?` 未知）。
- 解析前设默认值；解析后自己做**必填检查**。
- `shift $((OPTIND-1))` 之后处理剩余位置参数；短标志可合并 `-vf`。
- 错误/用法一律 `>&2`。
