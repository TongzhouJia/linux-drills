# KP B3.7 答案 — Error Handling & `trap`

> 全部真机跑过、输出核对无误。心法：**`trap '动作' 信号`**——`EXIT` 善后（无论怎么退都跑）、`ERR` 出错时触发、`INT/TERM` 接管中断。运维脚本的"安全气囊"。

---

**1 + 6 + 8.** `trap … EXIT` 自动清理（出错也跑）。
```bash
#!/bin/bash
set -e
tmp=$(mktemp)
cleanup(){ rm -f "$tmp"; echo "cleanup ran (removed $tmp)"; }
trap cleanup EXIT          # 用函数比内联字符串更清晰
echo "working with $tmp"
false                      # 失败 → set -e 触发退出
echo "NOT REACHED"
```
输出：`working with …` → `cleanup ran …`，退出码 1；临时文件已删。
*为什么：* **`EXIT` 陷阱在脚本以任何方式结束时都会执行**（正常结束、`exit`、`set -e` 中止、大多数信号）。所以把 `rm` 临时文件、释放锁、恢复现场都挂到 EXIT，绝不遗漏。`mktemp` 生成唯一临时文件名（别自己拼 `/tmp/foo.$$`）。

**2.** 临时目录。
```bash
d=$(mktemp -d)
trap 'rm -rf "$d"' EXIT
```
*为什么：* `mktemp -d` 建唯一临时目录，EXIT 里 `rm -rf` 整个删掉。下载、解压、中间产物都放里面，退出即净。

**3 / 9.** `trap … ERR` + 行号。
```bash
#!/bin/bash
set -e
trap 'echo "failed at line $LINENO (exit=$?)" >&2' ERR
true
false        # 第 5 行
# → failed at line 5 (exit=1)
```
*为什么：* `ERR` 陷阱在命令失败（返回非 0）时触发，配 `set -e` 用。`$LINENO` 给出出错行、`$?` 给出退出码——排障时直接知道**哪一行炸了**。生产脚本常这么记错误日志。

**4.** 接管 `SIGTERM`。
```bash
trap 'echo "caught SIGTERM, cleaning up"; exit 130' TERM
kill -TERM $$              # 给自己发 TERM
# → caught SIGTERM, cleaning up ; 退出码 130
```
*为什么：* `trap '动作' TERM`（或 `INT` 接 Ctrl-C）让脚本被中断时先善后再退。约定：被信号 N 终止时退出码 = `128+N`（TERM=15 → 143，但这里我们手动 `exit 130`）。长任务/守护脚本必备。

**5.** `die()` 助手。
```bash
die(){ echo "FATAL: $*" >&2; exit 1; }
[[ $# -ge 1 ]] || die "need an argument"
echo "got $1"
```
输出：无参 → `FATAL: need an argument`（stderr）、退出 1；有参 → `got …`。
*为什么：* 把"报错+退出"封成一个 `die`，全脚本复用，比到处写 `echo …>&2; exit 1` 干净。错误信息走 **stderr**。

**7.** 锁文件防并发。
```bash
LOCK=/tmp/myjob.lock
if ! mkdir "$LOCK" 2>/dev/null; then echo "already running, abort"; exit 1; fi
trap 'rmdir "$LOCK"' EXIT
echo "running exclusively"
```
输出：首次 → `running exclusively`；若锁已存在 → `already running, abort`，退出 1。
*为什么：* **`mkdir` 是原子操作**——同一时刻只有一个进程能建成同名目录，天然当锁用（比"检查文件存在再创建"没有竞态）。EXIT 陷阱负责释放锁。定时任务防止上一轮没跑完又启一轮，就用这招（或 `flock`）。

**10.** 忽略 / 恢复信号。
```bash
trap '' TERM        # 空动作 = 忽略该信号
kill -TERM $$       # 无效，脚本继续
echo "survived TERM"
trap - TERM         # 恢复默认处理
```
*为什么：* `trap '' 信号` = **忽略**（关键区段不想被打断时用）；`trap - 信号` = **恢复默认**。区分好：`trap '动作'`=自定义、`trap ''`=忽略、`trap -`=复位。

---

## 一句话总结（B3.7 核心）
- **`trap cleanup EXIT` + `mktemp`** = 临时文件/目录自动善后（任何退出路径都跑）——最该养成的习惯。
- **`trap '… $LINENO' ERR`**（配 `set -e`）出错即报行号；**`trap … INT/TERM`** 接管中断、退出码 `128+N`。
- **`die(){ echo "$*" >&2; exit 1; }`** 统一报错退出；**`mkdir` 锁**防并发。
- `trap ''`=忽略、`trap -`=复位。
