# KP B3.6 答案 — Arrays & Associative Arrays

> 全部真机跑过、输出核对无误。心法：**取全部一律 `"${arr[@]}"`（带引号！）；关联数组先 `declare -A`，键可以是任意字符串。**

---

**1.** 索引数组基础。
```bash
fruits=(apple banana cherry)
echo "${fruits[0]} ${#fruits[@]}"    # → apple 3
```
*为什么：* `(...)` 定义数组，`${arr[0]}` 取第 0 个（从 0 起），`${#arr[@]}` 是元素个数。

**2.** 追加 `+=`。
```bash
fruits+=(date); echo "${fruits[@]}"  # → apple banana cherry date
```
*为什么：* `arr+=(x)` 在末尾追加。可一次加多个 `arr+=(x y z)`。

**3.** 安全遍历。
```bash
for f in "${fruits[@]}"; do echo "$f"; done
```
*为什么：* **一定要写 `"${arr[@]}"`（双引号+@）**：这样每个元素是独立一项，带空格的元素也不会被拆开。`${arr[*]}` 会拼成一个字符串。

**4.** 所有下标 `${!arr[@]}`。
```bash
echo "${!fruits[@]}"                 # → 0 1 2 3
```
*为什么：* `${!arr[@]}` 取**键/下标**列表（普通数组是 0,1,2…；稀疏数组也能正确列出）。想连下标带值遍历：`for i in "${!arr[@]}"; do echo "$i:${arr[$i]}"; done`。

**5.** 切片。
```bash
echo "${fruits[@]:1:2}"              # → banana cherry
```
*为什么：* `${arr[@]:起:个数}` 取子数组。和字符串切片语法一致。

**6.** `mapfile` 把命令输出读进数组。
```bash
mapfile -t users < <(cut -d: -f1 passwd.sample)
echo "count=${#users[@]} first=${users[0]} last=${users[-1]}"
# → count=10 first=root last=dave
```
*为什么：* `mapfile -t`（=`readarray -t`）按行读入数组，`-t` 去掉每行末尾换行。`${users[-1]}` 是最后一个元素（负索引）。比 `while read` 攒数组更快。

**7.** 关联数组（字典）。
```bash
declare -A color
color[apple]=red; color[banana]=yellow
echo "${color[apple]}"               # → red
```
*为什么：* **必须先 `declare -A`**（否则 `color[apple]` 会被当算术下标）。键是字符串，`${color[键]}` 取值。

**8.** 遍历关联数组。
```bash
for k in "${!color[@]}"; do echo "$k=>${color[$k]}"; done | sort
# → apple=>red / banana=>yellow
```
*为什么：* `${!color[@]}` 取所有键。**关联数组遍历顺序不确定**（哈希序），要稳定输出就 `| sort`。

**9.** 关联数组当计数器（纯 bash 版 GROUP BY）。
```bash
declare -A c
while IFS=: read -r _ _ _ _ _ _ sh; do ((c[$sh]++)); done < passwd.sample
for s in "${!c[@]}"; do echo "$s ${c[$s]}"; done | sort
# → /bin/bash 4 · /bin/sh 1 · /sbin/nologin 2 · /usr/sbin/nologin 3
```
*为什么：* `((c[$sh]++))` 以 shell 路径为键累加——和 awk 的 `c[$7]++` 一个思路。不想为一点统计起 awk 时，纯 bash 也能分组计数。

**10.** 字符串拆成数组。
```bash
IFS=, read -ra arr <<< "a,b,c"
echo "${arr[1]} ${#arr[@]}"          # → b 3
```
*为什么：* `read -a`（或 `-ra`）把输入按 `IFS` 拆进数组。`IFS=,` 临时设逗号分隔。解析 CSV 单行、`PATH`（`IFS=:`）常用。

---

## 一句话总结（B3.6 核心）
- 索引数组：`arr=(...)`、`${arr[i]}`、全部 `"${arr[@]}"`、个数 `${#arr[@]}`、下标 `${!arr[@]}`、追加 `+=`、切片 `${arr[@]:i:n}`。
- 载入：`mapfile -t a < <(cmd)`（按行）、`read -ra a <<< "$s"`（按 IFS 拆）。
- 关联数组：**先 `declare -A`**，键任意字符串，遍历 `"${!m[@]}"`（顺序不定，配 `sort`）；`((m[$k]++))` = 纯 bash 计数器。
