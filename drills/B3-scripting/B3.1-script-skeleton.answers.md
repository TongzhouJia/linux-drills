# KP B3.1 答案 — Script Skeleton & Safe Header

> 全部真机跑过、输出核对无误。心法：**`set -euo pipefail` 是运维脚本的安全带**——出错即停、用未定义变量即报错、管道失败不放过。

---

**1.** shebang + 可执行。
```bash
cat > hi.sh <<'EOF'
#!/usr/bin/env bash
echo ok
EOF
chmod +x hi.sh && ./hi.sh          # → ok
```
*为什么：* 首行 `#!` 告诉系统用哪个解释器。`#!/usr/bin/env bash` 比写死 `#!/bin/bash` 更可移植（在 PATH 里找 bash）。没 `chmod +x` 就只能 `bash hi.sh`，不能 `./hi.sh`。

**2.** `set -e` 出错即停。
```bash
#!/bin/bash
set -e
false                 # 失败 → 脚本立即退出
echo SHOULD_NOT_PRINT # 不会执行
```
*为什么：* `-e` = 任何命令返回非 0 就终止脚本。防止"某步失败了后面还照跑"酿成更大事故。（例外：在 `if`/`&&`/`||` 里的命令不触发。）

**3.** `set -u` 禁用未定义变量。
```bash
#!/bin/bash
set -u
echo "v=$UNSETV"      # → 报错 UNSETV: unbound variable，退出
echo AFTER            # 不会执行
```
*为什么：* `-u` = 引用未定义变量直接报错，而不是当空串。能抓住拼错变量名、忘赋值这类隐蔽 bug。需要"可能为空"时用 `${VAR:-default}` 显式兜底。

**4.** `pipefail` 抓管道里的失败。
```bash
#!/bin/bash
set -eo pipefail
false | true          # 有 pipefail：管道判为失败 → 配 -e 退出
echo SHOULD_NOT_PRINT # 不会执行
```
*为什么：* 默认管道只看最后一段（B1.5）。`pipefail` 让"任一段失败=管道失败"，配 `-e` 才能真正在 `cmd | grep | awk` 链条里出错就停。

**5.** 安全头 + `usage()` 参数校验。
```bash
#!/bin/bash
set -euo pipefail
usage(){ echo "Usage: $(basename "$0") <name>" >&2; exit 1; }
[[ $# -eq 1 ]] || usage
echo "Hello, $1"
```
运行：无参 → `Usage: check.sh <name>`（stderr）、退出 1；`./check.sh Tongzhou` → `Hello, Tongzhou`。
*为什么：* `usage` 把用法打到 **stderr** 并 `exit 1`；`[[ $# -eq 1 ]] || usage` 是"参数不对就报用法"的标准写法。`$#` = 参数个数。

**6.** `IFS` 控制切分。
```bash
#!/bin/bash
IFS=: read -r a b c <<< "x:y:z"
echo "$a|$b|$c"       # → x|y|z
```
*为什么：* `IFS`（内部字段分隔符）决定 `read`/词拆分怎么切。临时 `IFS=: read` 只对这条命令生效。解析 `:` 分隔的 `/etc/passwd`、`PATH` 很常用。安全脚本常设 `IFS=$'\n\t'` 避免按空格乱切文件名。

**7.** `$0` 与 `basename`。
```bash
#!/bin/bash
echo "script: $(basename "$0")"   # ./t7.sh → script: t7.sh
```
*为什么：* `$0` 是脚本被调用的路径，`basename` 取最后的文件名。用来在用法/日志里显示脚本名，不暴露整条路径。

**8.** 完整起手模板（**建议以后每个脚本都从它开始**）。
```bash
#!/usr/bin/env bash
set -euo pipefail
IFS=$'\n\t'
usage(){ echo "Usage: $(basename "$0") <arg>" >&2; exit 1; }

main(){
  echo "processing $1"
}

[[ $# -ge 1 ]] || usage
main "$@"
```
运行：`./tpl.sh report.log` → `processing report.log`；无参 → 用法、退出 1。
*为什么：* 安全头 + IFS 加固 + usage 守卫 + `main "$@"` 封装。养成从这个骨架起步的习惯，脚本天然健壮、可读、好维护。

---

## 一句话总结（B3.1 核心）
- **每个脚本开头**：`#!/usr/bin/env bash` + `set -euo pipefail`（出错停 / 未定义变量报错 / 管道失败不放过）。
- **`usage()` → stderr + `exit 1`**，用 `[[ $# -eq N ]] || usage` 守卫参数。
- `IFS=$'\n\t'` 加固切分；`$(basename "$0")` 取脚本名；逻辑装进 `main`，末尾 `main "$@"`。
