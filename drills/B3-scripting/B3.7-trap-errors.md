# KP B3.7 — Error Handling & `trap` ⭐⭐⭐
**`trap … EXIT` 清理 · `mktemp` · `trap … ERR` · 信号 `INT/TERM` · `die()` · 退出码 · 锁文件 · `$LINENO`**

**10 tasks · ⭐⭐⭐ · RHEL 10 · server1 · write each as a real script and run it**

> 生产脚本的分水岭：**会不会善后。** `trap` 让你在脚本退出/出错/被中断时自动清理临时文件、释放锁、留下有用的错误信息。这一节值得多花时间。

---

**1.** Write a script that makes a temp file with `mktemp` and uses `trap … EXIT` to delete it automatically when the script ends.
*Acceptance:* prints the temp path; on exit the file is removed by the trap.

**2.** Same idea for a temp **directory** with `mktemp -d` and `rm -rf` in the EXIT trap.
*Acceptance:* dir exists during the run, removed on exit.

**3.** Add `trap '…' ERR` (with `set -e`) so a handler prints a message when any command fails.
*Acceptance:* `echo start; false; echo end` → prints `start`, the ERR message, and **not** `end`; exit non-zero.

**4.** Catch `SIGTERM` with a trap that cleans up and exits `130`. Trigger it by `kill -TERM $$`.
*Acceptance:* prints `caught SIGTERM, cleaning up`; the line after `kill` is never reached; exit `130`.

**5.** Write a `die()` helper that prints `FATAL: <msg>` to **stderr** and exits `1`; use it to require an argument.
*Acceptance:* no arg → `FATAL: need an argument` (stderr), exit 1; with arg → `got <arg>`, exit 0.

**6.** Show that `trap … EXIT` cleanup runs **even when the script aborts** on an error (combine with task 1's pattern + `set -e` + a failing command).
*Acceptance:* the cleanup message still prints and the temp file is gone, despite the abort.

**7.** Write a **lock-file** guard: `mkdir` a lock dir (atomic); if it already exists, print `already running, abort` and exit 1; otherwise `trap 'rmdir …' EXIT`.
*Acceptance:* first run → `running exclusively`; if the lock dir already exists → `already running, abort`, exit 1.

**8.** Write a proper cleanup **function** `cleanup()` and register it with `trap cleanup EXIT` (instead of an inline string).
*Acceptance:* `cleanup` runs once on any exit path.

**9.** Make an `ERR` trap report the **line number** with `$LINENO`.
*Acceptance:* `trap 'echo "failed at line $LINENO"' ERR` then a `false` on line 5 → `failed at line 5`.

**10.** Temporarily **ignore** `SIGTERM` with `trap '' TERM`, then **restore** the default with `trap - TERM`.
*Acceptance:* while ignored, `kill -TERM $$` does nothing (script continues); after `trap - TERM`, default behaviour is back.
