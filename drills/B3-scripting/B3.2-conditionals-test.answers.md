# KP B3.2 答案 — Conditional Tests

> 全部真机跑过、输出核对无误。心法：**字符串/文件用 `[[ ]]`，数值用 `(( ))`**；`=~` 做正则、`BASH_REMATCH` 抓分组。

---

**1.** 字符串相等。
```bash
f(){ [[ $1 == prod ]] && echo match || echo nomatch; }
f prod    # → match
f dev     # → nomatch
```
*为什么：* `[[ str == pat ]]`，右侧不加引号时是**通配匹配**（`prod*` 可模糊匹配）；要精确字面相等就给右侧加引号 `"prod"`。不等用 `!=`。

**2.** 空 / 非空。
```bash
x=""; [[ -z $x ]] && echo empty        # -z: 长度为 0
y="hi"; [[ -n $y ]] && echo nonempty   # -n: 长度非 0
```
*为什么：* 校验参数/变量是否给了值。`[[ ]]` 里变量不加引号也安全（不会因空值或空格出错）。

**3.** 数值比较。
```bash
n=150; (( n > 100 )) && echo big        # (( )) 里像数学一样写 > < == 
[[ 50 -gt 100 ]] && echo big2 || echo small2   # [[ ]] 里数值用 -gt/-lt/-eq...
```
*为什么：* 数值比较两种写法：`(( ))` 用 `> < >= == `（最直观）；`[[ ]]` 里必须用 `-gt -lt -ge -le -eq -ne`。**别在 `[[ ]]` 里用 `>` 比数字**——那会被当字符串比较/重定向。

**4.–5.** 文件与目录测试。
```bash
[[ -f /etc/hostname ]] && echo isfile   # -f: 存在且是普通文件
[[ -d /etc ]] && echo isdir             # -d: 存在且是目录
```
*为什么：* 常用文件测试：`-e` 存在、`-f` 普通文件、`-d` 目录、`-r/-w/-x` 可读/写/执行、`-s` 非空、`-L` 符号链接。动文件前先测，避免脚本炸。

**6.** 组合条件。
```bash
f=/etc/hostname
[[ -f $f && -r $f ]] && echo readable
```
*为什么：* `[[ ]]` 内部可直接用 `&&`、`||`、`!`。"既是文件又可读"再处理，是稳健脚本的习惯。

**7.** 正则匹配 `=~`。
```bash
for v in 123 12a; do
  [[ $v =~ ^[0-9]+$ ]] && echo "$v number" || echo "$v not-number"
done
# → 123 number / 12a not-number
```
*为什么：* `=~` 右边是**扩展正则**（不要给整个正则加引号，否则被当字面）。校验输入格式（纯数字、IP、邮箱）非常常用。

**8.** 抓分组 `BASH_REMATCH`。
```bash
d=2026-07-17
[[ $d =~ ^([0-9]+)-([0-9]+)-([0-9]+)$ ]] && \
  echo "y=${BASH_REMATCH[1]} m=${BASH_REMATCH[2]} d=${BASH_REMATCH[3]}"
# → y=2026 m=07 d=17
```
*为什么：* `=~` 匹配成功后，`BASH_REMATCH[0]` 是整个匹配、`[1] [2] …` 是各捕获组。**纯 bash 就能抽字段**，不用外部 sed/awk。

**9.** `[[ ]]` 里的通配匹配。
```bash
f=access.log
[[ $f == *.log ]] && echo logfile
```
*为什么：* `==` 右边不加引号时按 **glob** 匹配（`*.log`、`host??`）。判断扩展名/前缀特别顺手。

**10.** `[ ]` 的坑（为什么首选 `[[ ]]`）。
```bash
x=""
[ $x = y ]        # → bash: [: =: unary operator expected  (exit 2，崩了)
[[ $x == y ]]     # → 只是返回假 (exit 1)，不崩
```
*为什么：* `[ ]` 是外部命令，`$x` 为空时展开成 `[ = y ]` → 参数缺失报错；变量带空格同理。`[[ ]]` 是 bash 语法结构，不做词拆分、空值/空格都安全。**结论：脚本里一律用 `[[ ]]`**（除非要严格 POSIX 可移植才用 `[ ]` 并给变量加引号）。

---

## 一句话总结（B3.2 核心）
- **首选 `[[ ]]`**：字符串 `== != -z -n`、文件 `-f -d -r -s`、可含 `&& || !`、变量不加引号也安全。
- **数值**：`(( n > 100 ))` 最自然；在 `[[ ]]` 里必须 `-gt -lt -eq…`。
- **正则** `=~`（右侧不加引号），分组结果在 `${BASH_REMATCH[n]}`——纯 bash 抽字段。
- 记住第 10 题：`[ ]` 遇空值/空格会崩，`[[ ]]` 不会。
