# KP B3.2 — Conditional Tests
**`[[ ]]` vs `[ ]` · 字符串 `== != -z -n` · 数值 `-eq…`/`(( ))` · 文件 `-f -d -r…` · 正则 `=~` + `BASH_REMATCH`**

**10 tasks · ⭐⭐ · RHEL 10 · server1 · run each in bash and match the result**

> 判断是脚本的分叉点。**首选 `[[ ]]`**（bash 内建，更安全、支持 `=~`/`&&`/通配），数值比较用 `(( ))` 最自然。`[ ]` 是老式外部 test，坑多（见第 10 题）。

---

**1.** Write a function taking `$1`; print `match` if it equals `prod`, else `nomatch`.
*Acceptance:* `[[ $1 == prod ]]` → `prod`→`match`, `dev`→`nomatch`.

**2.** Check emptiness: print `empty` when a variable is empty (`-z`) and `nonempty` when it has a value (`-n`).
*Acceptance:* `[[ -z $x ]]` true for `x=""`; `[[ -n $y ]]` true for `y="hi"`.

**3.** Numeric compare two ways: `(( n > 100 ))` for `n=150`, and `[[ 50 -gt 100 ]]`.
*Acceptance:* first → `big`; second is false → `small2`.

**4.** Test that `/etc/hostname` is a regular file (`-f`).
*Acceptance:* `[[ -f /etc/hostname ]]` → `isfile`.

**5.** Test that `/etc` is a directory (`-d`).
*Acceptance:* `[[ -d /etc ]]` → `isdir`.

**6.** Combined test: a path is a regular file **and** readable (`-f` and `-r`) in one `[[ ]]`.
*Acceptance:* `[[ -f $f && -r $f ]]` → `readable`.

**7.** Regex `=~`: for each of `123` and `12a`, print whether it is all digits.
*Acceptance:* `[[ $v =~ ^[0-9]+$ ]]` → `123 number`, `12a not-number`.

**8.** Capture with `BASH_REMATCH`: match `2026-07-17` as `year-month-day` and print each part.
*Acceptance:* `[[ $d =~ ^([0-9]+)-([0-9]+)-([0-9]+)$ ]]` → `y=2026 m=07 d=17` (from `${BASH_REMATCH[1..3]}`).

**9.** Glob match inside `[[ ]]`: test whether `access.log` ends in `.log` using `==` with a pattern (no quotes on the pattern).
*Acceptance:* `[[ $f == *.log ]]` → `logfile`.

**10.** Show the `[ ]` pitfall: with an **empty** variable, `[ $x = y ]` breaks, but `[[ $x == y ]]` does not.
*Acceptance:* single-bracket errors with `[: =: unary operator expected` (exit 2); double-bracket just returns false (exit 1), no crash.
