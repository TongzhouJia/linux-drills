# KP B3.3 答案 — Loops & Reading Line-by-Line

> 全部真机跑过、输出核对无误。心法：**逐行读文件用 `while IFS= read -r line; do …; done < file`**，别用 `cat | while`。

---

**1.** `for` 遍历列表。
```bash
for s in alpha bravo charlie; do echo "Creating $s"; done
```
*为什么：* `for x in 列表` 最常用，列表可来自字面、`$(命令)`、`*.txt` 通配、`"$@"`。

**2.** C 风格 `for`。
```bash
for ((i=1;i<=5;i++)); do printf '%s ' "$i"; done; echo   # → 1 2 3 4 5
```
*为什么：* `for ((init; cond; step))` 适合"按次数/索引"循环，和 C 语言一致。

**3.** `for` + 花括号序列。
```bash
for i in {1..3}; do echo "node$i"; done
```
*为什么：* `{1..3}` 在循环里生成 `1 2 3`。注意 `{1..$n}` **不会**展开变量——要用变量上限就写 `$(seq 1 "$n")` 或 C 风格。

**4.** `while` + 计数器。
```bash
i=1; while (( i<=3 )); do echo "w$i"; ((i++)); done
```
*为什么：* 条件为真就继续。`(( ))` 做数值条件和自增最顺手。

**5.** `until`（直到成立）。
```bash
i=1; until (( i>3 )); do echo "u$i"; ((i++)); done
```
*为什么：* `until cond` = 条件**为假**时循环、为真时停，是 `while` 的反义。少用但偶尔更贴合语义（"直到成功为止"）。

**6.** 逐行读 + 按字段拆。
```bash
while IFS=: read -r u _ uid _; do echo "$u:$uid"; done < passwd.sample
# → root:0 / daemon:1 / bin:2 ...
```
*为什么：* `IFS=:` 让 `read` 按 `:` 切；`read -r u _ uid _` 把第 1 列给 `u`、第 3 列给 `uid`，其余用 `_` 丢弃。`-r` 防止反斜杠被吃掉。`< file` 从文件喂入。

**7.** `IFS= read -r` 保留空白。
```bash
printf '  indented\nplain\n' | while IFS= read -r line; do echo "[$line]"; done
# → [  indented]   [plain]
```
*为什么：* **把 `IFS` 清空**（`IFS=`）能保留每行的前导/尾随空格，否则会被裁掉。逐行原样处理（如带缩进的配置）必须这么写。

**8.** ⚠️ 管道子shell坑。
```bash
count=0; printf 'a\nb\nc\n' | while read -r x; do ((count++)); done; echo "$count"   # → 0
count=0; while read -r x; do ((count++)); done < <(printf 'a\nb\nc\n'); echo "$count"  # → 3
```
*为什么（重要）：* `cmd | while …` 里，管道右侧在**子shell**执行，循环里改的变量**出了循环就没了**（所以 count=0）。改用**重定向** `done < file` 或**进程替换** `done < <(cmd)`，循环就在当前 shell 跑，变量保留。踩过这坑的人都记一辈子。

**9.** 遍历文件（开 `nullglob`）。
```bash
shopt -s nullglob
for f in *.txt; do echo "file: $f"; done
```
*为什么：* 默认无匹配时 `*.txt` 会**原样**当成一个"文件名"循环一次（B1.2 的坑）。开 `nullglob` 后无匹配就不进循环，安全。

**10.** `break` / `continue`。
```bash
for i in 1 2 3 4 5; do
  [[ $i -eq 3 ]] && continue   # 跳过本次
  [[ $i -eq 5 ]] && break      # 整个跳出
  echo "$i"
done   # → 1 2 4
```
*为什么：* `continue` 跳到下一次迭代，`break` 直接结束循环。批处理里"跳过坏数据/满足条件就停"常用。

---

## 一句话总结（B3.3 核心）
- **`for`**：列表 / C 风格 `for ((;;))` / `{a..b}`；**`while`**（真则继续）/ **`until`**（假则继续）。
- **逐行读文件**黄金写法：`while IFS= read -r line; do …; done < file`（`IFS=` 保空白、`-r` 防转义、`< file` 不用 `cat|`）。
- **坑**：`cmd | while` 在子shell，循环里改的变量丢失 → 用 `done < file` 或 `done < <(cmd)`。
- 遍历文件开 `nullglob`；`break`/`continue` 控流程。
