# 出题计划与进度（RHCSA EX200 v10 + 数据中心运维 Bash）

> 一个仓库，两条线：**RHCSA 考纲的 8 个领域** + **把原考纲里那个只有 3 个知识点的 Shell Scripts 换成一整套 Bash**（`02-bash/`，6 个子领域 33 个 KP）。
>
> 滚动出题：说一个编号，我出那一批题；做完不复用，下次是新题。

---

## 一、规则

- **语言**：题目全用英文（跟真实考试、跟工单一致）。
- **不重复**：每题参数随机化（用户名、大小、路径、UID、端口、权限位都不同），同一 KP 的题互不雷同。
- **每题格式**：一句话任务（英文）+ 明确的验收点。答案我另给，**不进仓库**——仓库里只留题目和你自己写的东西。
- **节奏**：一次出一个 KP（约 6–12 题）。⭐⭐⭐ 的可能拆两批。

## 二、目录约定

每个 KP 一个文件夹，里面两个文件：

```
01-essential-tools/D1.1-io-redirection/
├── D1.1-io-redirection.md   ← 题目（我出）
└── D1.1-io-redirection.sh   ← 你的练习记录（做完才有）
```

`.sh` 存在 = 这个 KP 练过了。`ls` 一眼看进度。

## 三、环境

- 练习机 `bash-practice`（192.168.89.10，CentOS Stream 10，root 登录），仓库在 `/home/bash-practice/bash-practice`。
- ⚠️ **单机限制**：领域 04/05/07/09（存储、文件系统、网络、SELinux）共 23 个 KP 需要双机 + 多块磁盘，这台机器做不了。题目照出、照存，实操等环境到位。
- `samples/` 放题目要用的样例数据，`setup/bootstrap.sh` 是虚机重装后的一键恢复，`lib/skeleton.sh` 是脚本骨架。

### 02 · Bash（33 KP）⭐ 主线

原 RHCSA 领域 2 只有 3 个知识点，太浅。换成这套面向数据中心运维岗的完整 Bash 训练。
**纯 Bash**：不学 Ansible、不用 Python；唯一的非 Bash 工具是 `jq`，标为可选。

**验收线**：对着空文件，从零写出一个 30–60 行、带参数、`set -euo pipefail` 的安全脚本，解决真实运维任务——例如"扫 `/var/log`，统计最近一小时造成 5xx 最多的 10 个 IP，超阈值就告警"——同时能读懂、能调别人写的脚本。

三条高压线：**awk 性价比最高**（B2.4–2.6，日志分析八成靠它）、**健壮性**（`set -euo pipefail`、`trap`、引号、`[[ ]]`）、**读并修 bug**（面试高频，单列 B6.2）。
