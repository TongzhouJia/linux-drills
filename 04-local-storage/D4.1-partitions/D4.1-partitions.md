# KP D4.1 — Partitions on MBR & GPT disks (fdisk / parted)
**8 tasks · RHEL 10 · ⭐⭐⭐ server2 as root · add spare disks /dev/sdb /dev/sdc to the VM first**

**1.** On `/dev/sdb`, create a new **1 GiB primary partition** using `fdisk`, write the changes, and confirm the new `/dev/sdb1` appears with `lsblk`.

**2.** On `/dev/sdc`, create a **GPT** partition table and add a **2 GiB** partition using `parted`.

**3.** Format the new `/dev/sdb1` partition as **ext4**.

**4.** Format the new `/dev/sdc1` partition as **xfs**.

**5.** Create another partition on `/dev/sdb` and set its partition **type to "Linux LVM"** (type code `8e` in fdisk, or the `lvm` flag in parted).

**6.** Mount `/dev/sdb1` at `/mnt/data1` and make it **persistent by UUID** in `/etc/fstab`; verify with `mount -a` (it must not error).

**7.** Delete the LVM-type partition you created in task 5 using `fdisk`, and confirm it is gone.

**8.** After creating/removing partitions **without rebooting**, force the kernel to re-read the partition table so the device nodes update, and verify with `lsblk`.
