# KP D9.8 — SELinux Troubleshooting
**7 tasks · RHEL 10 · ⭐⭐⭐ server1 as root · SELinux enforcing**

**1.** A service is being blocked. Temporarily switch SELinux to **permissive** to confirm SELinux is the cause, then switch back to enforcing.

**2.** Find the most recent **AVC denials** and save them to `/root/avc.txt` (use `ausearch`).

**3.** Install `setroubleshoot-server` and use `sealert` to analyze `/var/log/audit/audit.log` for a **fix suggestion**.

**4.** Content restored from backup into a service directory is inaccessible. Diagnose that it is a **context** problem and fix it with `restorecon`.

**5.** Follow the audit log **live** and reproduce a denial (e.g. `tail -f /var/log/audit/audit.log | grep AVC`).

**6.** Given an AVC denial that points to a wrong **file type**, fix it **persistently** (`semanage fcontext` + `restorecon`).

**7.** Given an AVC denial that a **boolean** would resolve, identify and set the correct boolean.
