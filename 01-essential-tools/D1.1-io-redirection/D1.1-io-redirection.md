# KP D1.1 — Input/Output Redirection & Pipes
**Batch 1 · 6 tasks · RHEL 10 · perform on server1 as root unless stated**

> Answers are kept separately. Do each task, then verify the resulting file yourself. No text editor allowed for building the files — use redirection/pipes only.

---

**1.** Write the output of `uname -r` into a **new** file `/root/kernel.txt` (overwrite if it exists). Then, **without erasing** what's already there, append the output of `hostname` and then the output of `date` to the same file, so `/root/kernel.txt` ends up holding all three lines in that order.

**2.** Run `ls -l /etc /does-not-exist /var` as a single command so that the normal directory listing is saved to `/root/good.txt` while the error message about the missing directory is saved to a **separate** file `/root/bad.txt`. Nothing should be printed to your terminal.

**3.** Run `find /etc -name "*.conf"` so that **both** its normal output **and** its permission-denied errors are written **together, combined, into one** file `/root/find_all.txt`, with nothing shown on screen.

**4.** Run the same `find /etc -name "*.conf"` but this time keep the normal results on screen and **silently discard all error messages** (throw them away, don't save them anywhere).

**5.** Using a **pipeline** (no editor), create `/root/shells.txt` containing an **alphabetically sorted, duplicate-free** list of every login shell currently assigned in `/etc/passwd` (that's the last `:`-separated field of each line).

**6.** As the **regular (non-root) user**, append the output of `journalctl -k -n 20` to the root-owned file `/var/log/kmsg_snap.log`, creating it if needed. You must use `sudo` — note that a plain `sudo journalctl ... >> /var/log/kmsg_snap.log` will fail, so make the append actually work.
