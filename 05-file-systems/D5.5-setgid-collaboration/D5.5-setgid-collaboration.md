# KP D5.5 — set-GID Directories for Collaboration
**6 tasks · RHEL 10 · server1 as root · group `dba_staff` should already exist**

**1.** Create a shared directory `/shared/dba`, set its **group owner** to `dba_staff`, and set the **set-GID** bit so new files inside inherit the `dba_staff` group.

**2.** Set permissions so only the owner and `dba_staff` members can access it (no access for others) — i.e. mode `2770`.

**3.** Add the **sticky bit** so users can only delete files they own, even inside the shared directory.

**4.** Verify with `ls -ld` that both the setgid (`s`) and sticky (`t`) bits are shown, and save that line to `/root/collab_check.txt`.

**5.** As a `dba_staff` member, create a file in the directory and confirm it **inherits** group `dba_staff`.

**6.** Create a single directory `/shared/project` that satisfies **all** at once: group-owned by `dba_staff`, set-GID, sticky, and accessible only to that group — using one `chmod` numeric mode.
